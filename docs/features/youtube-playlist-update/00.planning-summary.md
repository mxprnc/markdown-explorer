# [Feature Planning Summary] Youtube Playlist Update

**Feature Name**: `youtube-playlist-update`
**Date**: 2026-05-04
**Status**: Planning Completed

## 1. 개요 (Overview)
- **문제 정의**: 기존 "YouTube Playlist Export" 기능 사용 시, 사용자가 추가해 둔 메모나 일반 텍스트가 섞여 있을 때 플레이리스트(영상 목록)만 깔끔하게 업데이트(재정렬, 삭제, 추가)하기 어려운 문제 발생.
- **목표**: 마크다운 문법을 훼손하지 않으면서, 시스템이 명확하게 파싱할 수 있고 사용자 노트가 안전하게 결합(Binding)된 새로운 구조를 기획하여 업데이트 안정성과 사용자 경험을 향상시킨다.

## 2. Product Owner (기획 핵심 결정)
**최종 채택안: Export 이원화 전략 (Dual Export Strategy)**
사용자의 다양한 니즈를 충족시키기 위해 Export 기능을 두 가지 모드로 분리하고, 각각에 최적화된 마크다운 템플릿과 업데이트 파서를 제공합니다.

1. **Youtube Playlist Markdown (전용 문서 모드)**
   - **구조**: `## 제목` + `링크` + `### Note` (D-1 방식)
   - **목적**: 해당 재생목록 전용 노트를 만들고 싶을 때 사용. 영상별로 큰 챕터가 나뉘는 굵직한 구조.
2. **Playlist Link List (기존 문서 혼합 삽입 모드)**
   - **구조**: `- **제목**` + `링크` + `> 메모영역` (D-2 방식)
   - **목적**: 사용자가 다른 주제로 글을 쓰다가 레퍼런스로 재생목록을 살짝 끼워 넣고 싶을 때 사용. 목차(TOC)를 오염시키지 않고 깔끔하게 삽입됨.

## 3. UX Design (디자인 핵심 결정)
**최종 채택안: 우측 사이드 패널 (Slide-over Drawer) & 터치 최적화**
- **문맥 유지**: 중앙을 가리는 모달 대신 우측에서 나오는 슬라이드 서랍(Drawer) 형태를 채택하여, 사용자가 본문의 텍스트를 계속 보면서 업데이트할 수 있도록 배려함.
- **프리미엄 비주얼**: `Zine 900` 다크 배경에 `Indigo to Violet` 그라데이션을 포인트로 사용하여 고급스러운 무드 연출.
- **조작 최적화**: 마우스 기반의 Drag & Drop 뿐만 아니라, 태블릿/스마트폰 터치 환경에서도 쉽게 다룰 수 있도록 리스트 아이템 우측에 명시적인 [위/아래 화살표] 버튼을 제공. 조작 시 미세한 팝업(Scale) 및 스왑 애니메이션 적용.

## 4. Implementation Feasibility (기술 검토)
- **결과**: **구현 가능 (Yes)**
- **사유**: 
  - 마크다운 파싱 시 정규식(Regex) 또는 AST(Abstract Syntax Tree)를 활용하여 D-1, D-2 구조를 명확히 분리/합합할 수 있음.
  - UI 측면에서 React Portal과 Framer Motion(또는 유사 애니메이션 라이브러리)을 이용해 Drawer 및 리스트 애니메이션을 무리 없이 구현 가능함.

## 5. Update Action Logic (업데이트 동작 로직)
**최종 채택안: Idea G (Auto Diff & Trash) + 에디터 API 연동**
- **스마트 동기화 (Auto Diff)**: 우측 패널(모달) 오픈 시 백그라운드에서 YouTube API를 호출하여, 현재 문서에 없는 '새로운 영상'이 발견되면 리스트에 자동으로 추가해 보여줍니다.
- **삭제 동기화 (Trash)**: 패널 안에서 특정 영상을 삭제(또는 숨김 처리)하고 '적용'을 누르면, 마크다운 본문에서도 해당 영상의 URL과 **사용자가 작성한 노트(메모)** 영역이 함께 깔끔하게 삭제됩니다.
- **안전망 (Cmd + Z 완벽 지원)**: 데이터를 파일(Disk)에 직접 덮어쓰는 것이 아니라, Mark Explorer의 내장 에디터 컴포넌트(상태)를 제어하는 API를 통해 업데이트 텍스트를 주입합니다. 이를 통해 사용자가 실수로 노트를 지웠더라도 언제든지 **`Cmd + Z (Undo)`**를 눌러 이전 상태로 완벽하게 롤백할 수 있는 강력한 안전망을 제공합니다.

## 6. [Round 2] 모달 고도화 및 진입점 확장 (2026-05-04 추가)
- **Live Fetch Preview**: 중앙 Export 다이얼로그를 50:50(좌우)으로 넓혀, URL 입력 후 실제 영상 1~2개를 바탕으로 현재 선택된 모드(D-1/D-2)가 어떻게 텍스트로 추출될지 우측에 실시간 렌더링(Markdown / Preview 토글 제공)해 줍니다.
- **탐색기 Context Menu 연동**: 탐색기 사이드바에서 특정 폴더를 우클릭하여 `Create Youtube Playlist Note`를 누르면, 즉시 D-1 모드로 고정된 다이얼로그가 열리며 [Extract] 버튼 클릭 시 해당 폴더에 파일이 곧바로 생성되는 빠른 파이프라인을 구축합니다.

## 7. [Round 3] 구현 간 이슈 트래킹 및 버그 수정 (2026-05-04 추가)
- **D-1 파일 생성 및 포맷 스펙 복구**: 
  - 누락되었던 D-1 전용 모드 파일명 규격(`{재생목록 제목}_{HHmmss}.md`)과 각 아이템 내부 포맷(`Text | URL | Card | Video`) 유지 결정을 기획 및 구현에 동기화함.
- **포맷 프리뷰(Preview) 렌더링 오류 해결 (mx-thumb / mx-video)**:
  - **현상**: `Card` 포맷이 `[mx-link]`로 오파싱되거나, 모달의 가짜(Mock) URL(`v=mock1`)이 11자리 YouTube ID 정규식 조건을 만족하지 못해 비디오 플레이어 및 카드가 렌더링되지 않고 일반 파란색 링크로 폴백(Fallback)됨.
  - **해결**: 파서 유틸리티를 `mx-thumb`으로 교체하고, `MarkdownPreview` 렌더러(웹/네이티브 공통)에 썸네일 카드 UI 지원 로직을 추가함. Mock URL 역시 `dQw4w9WgXcQ` 등 11자리로 교체하여 프리뷰를 완벽히 정상화함.
- **`format is not a function` 런타임 에러 해결**:
  - **현상**: `app/index.tsx`에서 파일 생성 시 `date-fns`의 `format`을 호출하다가 예외 발생.
  - **해결**: 외부 라이브러리에 의존하지 않고, 순수 JavaScript `new Date()`를 사용한 문자열 조작 로직(HHmmss 생성)으로 교체하여 에러를 원천 차단함.
- **D-1 파일 생성 시 빈 탭이 열리는 현상 수정**:
  - **현상**: `Extract & Create File` 클릭 시 파일은 디스크에 저장되나, 사이드바 탐색기(React State)에 갱신되지 않은 상태에서 해당 탭을 열어 빈 화면(`''`)이 노출됨.
  - **해결**: `handleSaveToDiskInHook` 호출 직전에 `createItem` API를 호출하여 메모리 상의 `fileSystemData` 상태를 먼저 동기화한 후, 내용을 기록하고 탭을 열도록 라이프사이클을 수정함.
- **가짜(Mock) 데이터 제거 및 실제 YouTube API 연동 완료**:
  - **현상**: 모달 내의 Live Preview 및 최종 파일 추출 시 하드코딩된 'Sample Playlist' 및 목업(Mock) 영상 2개만 반환됨. 파일명도 무조건 `Sample_Playlist_{HHmmss}.md`로 고정됨.
  - **해결**: `YoutubeUtils.ts`의 `fetchPlaylistItems`와 `fetchPlaylistMetadata` 함수를 호출하도록 코드를 리팩토링함. 이를 통해 실제 YouTube 재생목록의 메타데이터(제목)와 영상 항목들을 동적으로 불러오며, 파일명 역시 `{실제 플레이리스트 제목}_{HHmmss}.md` 로 정확히 동적으로 치환되도록 수정함.

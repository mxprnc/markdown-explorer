# Split View (분할 창) 기능 명세

Split View는 하나의 에디터 영역을 두 개의 창(Pane)으로 나누어, 서로 다른 두 파일을 동시에 보거나 편집할 수 있게 해주는 기능입니다. VSCode와 같은 현대적인 에디터의 사용자 경험(UX)을 지향합니다.

## 🚀 주요 기능

### 1. 분할 창 토글 (Toggle Split View)
- **UI 위치**: 헤더(Header) 우측 상단의 'Split View' 버튼.
- **동작**:
  - 버튼 클릭 시 에디터 영역이 가로로 2등분되어 Pane 1(좌)과 Pane 2(우)가 나타납니다.
  - 다시 클릭하면 Pane 2가 닫히고 Pane 1이 전체 영역을 차지합니다.
  - Pane 2를 닫을 때, Pane 2에서 열려 있던 파일들의 상태(Opened Files)는 보존되지만 화면에서는 사라집니다.

### 2. 활성 창 관리 (Active Pane)
- **포커스 감지**: 클릭하거나 파일을 선택한 창이 '활성 창'이 됩니다.
- **시각적 피드백**: 활성 창은 테두리가 강조되거나 탭 바의 스타일이 변화하여 사용자에게 현재 포커스 위치를 알립니다.
- **Explorer 연동**: 파일 탐색기(Explorer)에서 파일을 클릭하면 현재 **활성 상태인 Pane**에 해당 파일이 열립니다.

### 3. 탭 드래그 앤 드롭 (Tab Drag & Drop)
- **Pane 내 재정렬**: 동일한 Pane 안에서 탭을 드래그하여 순서를 자유롭게 바꿀 수 있습니다.
- **Pane 간 이동**: 탭을 드래그하여 다른 Pane으로 옮길 수 있습니다.
- **삽입 위치 표시**: 드래그 중인 탭이 놓일 위치에 세로선(Insertion Indicator)이 표시되어 드롭 결과를 예측할 수 있게 합니다.

## 🛠 기술적 세부 사항

- **상태 분리**: `openedFiles1`, `selectedFile1` / `openedFiles2`, `selectedFile2`와 같이 각 Pane의 파일 목록 및 선택 상태가 독립적으로 관리됩니다.
- **데이터 보존**: 드래그 앤 드롭 시 파일의 경로뿐만 아니라 소스 Pane 정보가 함께 전달되어 상태 배칭(Batching) 시 데이터 유실을 방지합니다.

## 🔴 제약 사항 및 미해결 이슈

### 1. 빈 Pane 드롭 불가 (Known Bug)
- **상황**: Pane 2에 열린 파일이 하나도 없을 때(Empty State), Pane 1의 탭을 Pane 2로 드래그하여 드롭해도 이동되지 않는 현상이 있습니다.
- **우회 방법**: Explorer에서 파일을 하나 이상 선택하여 Pane 2를 먼저 채운 후에는 탭 이동 기능이 정상 작동합니다.
- **상세 내용**: [issue-20260420.md](./errors/split-view---editor-tab-drag-drop/issue-20260420.md) 참조.

## 📅 향후 개선 계획

- **Pane 병합**: 분할 창 모드 종료 시 Pane 2의 탭들을 Pane 1로 자동으로 합치는 기능 추가 예정.
- **중첩 분할**: 상하 분할 및 3개 이상의 다중 창 분할 지원 검토.
- **레이아웃 보존**: 앱 재시작 시 마지막 분할 상태 및 각 Pane의 파일 목록 복원 기능.

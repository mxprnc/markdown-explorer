# [Issue] Option + T (템플릿 삽입) 단축키 작동 불일치

**상태**: 🔴 해결 중 (In Progress)  
**날짜**: 2026-04-21  
**환경**: Mac OS (Apple Silicon), Chrome/Edge Browser (Web)

---

## 🔍 증상 (Symptoms)
1. 에디터 내에서 타이핑 중 `Option (⌥) + T`를 눌러도 템플릿 선택기가 나타나지 않음.
2. 특정 상황(왼쪽 Explorer의 `.mark-explorer/templates` 디렉터리에 마우스 포인터가 있을 때 등)에서만 간헐적으로 작동함.
3. 에디터 포커스 유무에 따라 반응이 달라지는 현상이 관찰됨.

## 🛠 시도된 해결책 (Attempted Solutions)
1. **단축키 매핑 수정**: `key` 대신 물리적 키 위치인 `code ('KeyT')`를 사용하여 Mac OS의 특수 문자 입력 간섭 회피.
2. **에디터 내 단축키 등록**: Tiptap Extension을 통해 에디터 자체 단축키로 등록 시도.
3. **이벤트 캡처링 적용**: `window.addEventListener`에 `{ capture: true }` 옵션을 추가하여 하위 컴포넌트의 이벤트 차단 방지.

## ❓ 원인 분석 (Root Cause Analysis - 가설)
### 1. Mac OS의 Option 키 자판 레이아웃 간섭
Mac에서 `Option` 키는 특수 문자 입력을 위해 사용되므로, 브라우저가 `keydown` 이벤트 시점에 `e.key`를 변경하거나 기본 동작을 선점하여 React Native/Tiptap 레벨까지 전달되지 않을 수 있습니다.

### 2. 컴포넌트별 이벤트 전파 차단 (Stop Propagation)
Tiptap(ProseMirror) 에디터나 Explorer의 특정 라이브러리가 이벤트를 가로채어(`e.stopPropagation()`) 전역 리스너(`window`)까지 도달하지 못하게 할 가능성이 큽니다.

### 3. 호버/포커스 컨텍스트 문제
사용자가 보고한 "특정 폴더 호버 시 작동" 현상은, 해당 영역이 마우스 오버 시 일시적으로 포커스를 가져가거나 브라우저의 이벤트 수신 우선순위가 변경되면서 단축키가 동작하는 것으로 추정됩니다.

## 🚀 향후 작업 계획 (Next Steps)
- [ ] **전역 핫키 라이브러리 검토**: `react-hotkeys-hook` 등 브라우저 간섭을 효과적으로 처리하는 검증된 라이브러리 도입 검토.
- [ ] **커스텀 단축키 설정 UI**: 사용자가 직접 단축키를 변경하여 Mac OS 기본 단축키와 충돌하지 않게 조치.
- [ ] **에디터 명령 센터 통합**: 에디터 내에서 `/` (Slash Command)를 통해 템플릿을 선택하는 대체 UI 제공 (단축키 의존도 감소).

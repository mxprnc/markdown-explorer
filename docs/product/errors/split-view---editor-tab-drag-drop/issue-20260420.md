# Issue: Spliev View - Editor Tab Drag & Drop Failure on Empty Pane

## 📅 발생 일자
2026-04-20

## 🔴 이슈 요약
Split View 모드에서 한쪽 Pane(주로 Pane 2)이 비어 있을 때, 다른 Pane에서 탭을 드래그하여 비어 있는 영역에 드롭해도 파일이 이동되거나 열리지 않는 현상.

## 🔎 상세 증상
1. Pane 1에 여러 파일이 열려 있고 Pane 2가 비어 있는 상태에서 'Split View' 활성화.
2. Pane 1의 탭 하나를 잡아 Pane 2의 빈 공간(Editor Area)으로 드래그.
3. 브라우저의 드래그 아이콘(+)이 나타나지만, 드롭하는 순간 아무런 반응이 없음.
4. Pane 2에 최소 하나 이상의 파일이 열려 있는 상태에서는 정상적으로 드롭 및 탭 순서 변경이 작동함.

## 🛠 시도된 해결 방법 및 한계
- **데이터 포맷 정규화**: `dataTransfer`에 소스 정보를 포함했으나 특정 브라우저 환경에서 유실 가능성 확인.
- **전면 드롭 레이어(Overlay)**: 빈 공간 전체를 덮는 레이어를 추가했으나 플리커링 및 이벤트 전파 문제로 완벽한 해결 실패.
- **드래그 카운터**: 중첩된 요소 간의 이벤트를 관리하려 했으나 구조적 복잡성 증가.

## 📅 향후 계획
본 프로젝트의 초기 안정화 단계에서는 **'빈 Pane은 Explorer에서 파일을 선택하여 먼저 채우는 방식'**을 권장하며, 해당 드래그 기능은 추후 '미해결 이슈 수정 및 최적화' 기간에 브라우저 네이티브 이벤트를 전면 재검토하여 수정할 예정입니다.

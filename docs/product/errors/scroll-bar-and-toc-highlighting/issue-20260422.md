# Issue Report: TOC Highlight Flickering during Programmatic Scroll

## 🗓 Date: 2026-04-22
## 👤 Status: Identified / Reporting

---

## 🔍 Description
TOC(목차) 내비게이션에서 특정 섹션을 클릭하여 이동할 때, 하이라이트 표시가 클릭한 목표 섹션에 고정되지 않고 스크롤 도중 지나치는 중간 섹션들을 하나씩 거쳐가는 '점프' 현상이 발생합니다. 사용자가 목표한 지점으로 이동하는 과정이 시각적으로 산만하게 느껴질 수 있습니다.

## 🔴 Root Cause (원인 분석)
두 가지 상태 업데이트 로직 간의 충돌로 인해 발생합니다:
1.  **Programmatic Update**: TOC 클릭 시 즉시 `setActiveHeadingIndex`를 호출하여 목표 인덱스로 하이라이트를 변경합니다.
2.  **Scroll Spy (Intersection Observer)**: 클릭 이후 `smooth scroll`이 진행되는 동안, 화면 중앙/상단을 지나치는 모든 중간 헤딩들이 `IntersectionObserver`에 의해 감지됩니다.
3.  **Conflict**: 옵저버가 중간 섹션들을 감지할 때마다 `onHeadingVisible`을 호출하여 전역 상태인 `activeHeadingIndex`를 계속 덮어쓰게 되어, 하이라이트가 스크롤을 따라가는 현상이 나타납니다.

## 🛠 Proposed Fix (해결 제안)
*   **Scroll Locking**: TOC 클릭에 의한 스크롤 발생 시, 일시적으로 `IntersectionObserver`의 업데이트를 무시하는 플래그(예: `isProgrammaticScrolling`)를 도입합니다.
*   **Timeout / End Event**: 스크롤이 완료된 시점(또는 일정 시간 후)에 해당 플래그를 해제하여 다시 자동 감지 모드로 복귀하도록 구현합니다.


## 🎬 Reproduction Steps (재현 단계)
1. 여러 개의 섹션이 있는 긴 마크다운 문서를 엽니다.
2. 현재 위치에서 멀리 떨어진 TOC 항목을 클릭합니다.
3. 문서가 부드럽게 스크롤되는 동안 TOC의 하이라이트 막대가 중간 섹션들을 하나씩 거쳐 목표 지점으로 이동하는 것을 확인합니다.

## 🛠 Proposed Fix (해결 제안)

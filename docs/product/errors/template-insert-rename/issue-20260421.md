# Issue Report: Template Insertion & Renaming (2026-04-21)

## 📌 현상 요약
템플릿 전용 UI 구축 후 핵심 기능인 **삽입(Insert)**과 **이름 변경(Renaming)** 기능이 정상적으로 동작하지 않는 문제가 발생함.

## 🔍 상세 증상 및 분석

### 1. 템플릿 삽입(Insert) 기능 미동작
- **증상**: 사이드바의 템플릿 리스트에서 `Insert` 버튼을 클릭해도 에디터 본문에 내용이 추가되지 않음.
- **분석된 원인 후보**:
    - **이벤트 전파 이슈**: `Pressable` 컴포넌트 내의 버튼 클릭 시 `stopPropagation` 처리가 웹 환경(React Native Web)에서 의도대로 작동하지 않아 메인 행 클릭 기능(Edit)과 충돌 가능성.
    - **에디터 참조(Ref) 누락**: 파일을 새로 열거나 탭을 전환할 때 Tiptap 에디터 인스턴스가 완전히 렌더링되기 전에 삽입 명령이 전달되어 `targetRef.current`가 null인 상태일 수 있음.
    - **파일 시스템 경로 불일치**: 숨김 폴더(`_mark-explorer/templates`) 경로 접근 시 `Vault` API가 브라우저의 파일 시스템 핸들을 안정적으로 유지하지 못할 가능성.

### 2. 템플릿 제목 변경(Renaming) 기능 부재/미동작
- **증상**: 템플릿 리스트에서 파일명을 직접 수정할 수 있는 UI나 기능이 아직 구현되지 않았거나, 내부적으로 파일명 변경 명령이 실패함.
- **분석된 원인 후보**:
    - 현재 템플릿 시스템은 `Edit`을 통해 파일 내용을 수정하도록 설계되었으나, 파일명 자체를 변경하는 전용 명령어가 `TemplatesPlugin`에 정의되지 않음.

## 🛠 대응 현황 및 계획
- [x] **디버깅 코드 주입**: `Vault.read` 및 `onInsertText` 단계에 알림(Alert)을 추가하여 데이터 흐름 추적 중.
- [x] **강력한 파일 로딩 로직 도입**: 트리 캐시(State)가 아닌 물리 디스크 핸들을 직접 순회하는 폴백 로직 적용 완료.
- [ ] **구조 재설계**: 웹 환경에서 안정적인 이벤트 버스(Event Bus) 동작을 보장하기 위해 리스너 등록 시점과 에디터 마운트 시점의 동기화 개선 필요.
- [ ] **파일명 변경 기능 추가**: 사이드바에서 즉시 이름을 변경할 수 있는 인라인 편집 기능 개발 예정.

## 📅 후속 작업
- 전역 이벤트 리스너를 상태 기반이 아닌 고정된 핸들러 방식으로 리팩토링.
- 템플릿 폴더 전용 `create/rename/delete` 명령어를 코어에 정식 등록.

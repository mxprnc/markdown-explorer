# Editor Link Card 트러블슈팅 리포트 (Troubleshooting)

이 문서는 Link Card 기능을 개발하면서 직면했던 기술적 문제들과 그에 대한 해결 과정을 기록합니다.

---

## 1. 유튜브 비디오 깜빡임 및 재로드 이슈 (Flickering)

### 증상
에디터 모드에서 타이핑을 할 때마다 익스플로러(미리보기) 모드에 띄워진 유튜브 비디오 카드가 "치지직"거리며 깜빡이거나 처음부터 다시 로드되는 현상 발생.

### 원인
- `EditorWorkspace.tsx`에서 `MarkdownPreview` 컴포넌트에 `resolveImage` 핸들러를 넘길 때 인라인 화살표 함수(`(src) => resolveImage(src, selectedFile)`)를 사용함.
- 타이핑할 때마다 `EditorWorkspace`가 리렌더링되면서 새로운 함수 참조가 전달되었고, 이로 인해 `MarkdownPreview` 내부의 `useMemo`로 관리되던 컴포넌트 맵이 매번 재생성됨.
- React는 컴포넌트 타입이 바뀌었다고 판단하여 기존 `iframe`을 언마운트하고 다시 마운트함.

### 해결 방법
- `EditorWorkspace.tsx` 내부에서 `resolveImage`를 `useMemo` 또는 `useCallback`으로 감싸 참조 안정성(Referential Stability)을 확보함.
- 파일이 바뀌지 않는 한 타이핑 중에는 동일한 함수 참조가 유지되어 불필요한 리렌더링 및 언마운트가 차단됨.

## 2. UI 모드 복구 실패 및 텍스트 고착 이슈

### 증상
링크 카드의 타입을 변경(예: Video -> Card)한 후, 텍스트 모드(`[mx-thumb#...](...)`)로 변한 상태에서 커서를 다른 곳으로 옮겨도 다시 UI 카드로 돌아오지 않고 텍스트 상태로 유지됨.

### 원인
- **위치 계산 오차**: `LiveMarkdownExtension`에서 텍스트 블록 내부의 좌표를 계산할 때, 블록 안에 이미 다른 노드(이미지 등)가 있으면 `index`가 밀려 정확한 패턴 위치를 잡지 못함.
- **노드 감지 실패**: `nodeAt(start)` 방식은 텍스트 노드 중간에서 `null`을 반환하는 경우가 있어, 이미 노드가 생성되었는지 판단하는 로직이 불안정했음.
- **정규식 엄격성**: 기존 정규식이 `#` 기호나 비어 있는 속성값에 대해 엄격하여 타입을 변경하는 과정에서의 미세한 텍스트 변화를 수용하지 못함.

### 해결 방법
- **좌표 계산 보정**: `pos + 1 + match.index` 방식을 사용하여 텍스트 블록 내 절대 좌표를 정확히 산출함.
- **감지 로직 강화**: `nodeAt` 대신 `nodesBetween`을 사용하여 해당 범위 내에 `linkCard` 노드가 이미 존재하는지 더 견고하게 체크함.
- **정규식 최적화**: `mxRegex`를 `/\[mx-(thumb|link|video|plain)#?([^\]]*)\]\(([^)]+)\)/g`로 업데이트하여 `#`를 선택적으로 처리하고 유연한 매칭을 지원함.

## 3. 기본 플레이스홀더 노출 문제 (Link Card / YouTube Video)

### 증상
링크 카드를 생성하고 제목(Alt Text)을 입력하지 않았을 때, 카드 UI 상단에 시스템 기본 텍스트인 'Link Card' 또는 'YouTube Video'가 강제로 노출되어 디자인적으로 지저분함.

### 원인
`LinkCardComponent.tsx`와 `MarkdownPreview.web.tsx` 양쪽 코드에 제목이 없을 경우의 폴백(Fallback) 문자열이 하드코딩되어 있었음.

### 해결 방법
- `getDerivedMetadata` 및 `a` 태그 매퍼에서 폴백 문자열을 제거하고 빈 문자열(`''`)을 반환하도록 수정함.
- 제목이 없을 경우 해당 영역이 공간을 차지하지 않거나 자연스러운 공백으로 보이도록 UI 조건부 렌더링을 적용함.

---

## 4. 실시간 마크다운 변환 루프 방지

### 증상
텍스트를 노드로 변환하고 다시 텍스트로 확장하는 과정에서 무한 루프나 트랜잭션 충돌 가능성.

### 해결 방법
- `evalMarkdown` 메타데이터를 트랜잭션에 추가하여, 이미 평가가 진행 중인 트랜잭션은 중복 처리를 하지 않도록 방어 로직을 구축함.
- `appendTransaction`을 사용하여 문서 변화나 선택 영역 변화가 있을 때만 신중하게 변환 로직을 수행함.

---

> [!TIP]
> 향후 유사한 인라인 노드나 시각적 카드를 추가할 때 이 트러블슈팅 리포트를 참조하여 참조 안정성과 정규식 유연성을 우선적으로 고려해야 합니다.

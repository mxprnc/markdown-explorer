# 플러그인 시스템 제품 명세 (plugin-system.md)

Mark Explorer는 사용자의 필요에 따라 기능을 확장하고 개인화할 수 있는 플러그인 기반 아키텍처를 제공합니다.

## 🌟 핵심 가치
- **무한한 확장성**: 코어 프로그램을 수정하지 않고도 새로운 기능을 추가할 수 있습니다.
- **개인화된 경험**: 필요한 기능만 활성화하여 가볍고 최적화된 작업 환경을 유지할 수 있습니다.
- **커뮤니티 중심**: 표준 인터페이스를 통해 누구나 자신만의 도구를 만들어 공유할 수 있습니다.

## 🛠 주요 기능
1. **플러그인 관리자 (Plugin Manager)**
   - 설치된 플러그인 목록을 확인하고, 실시간으로 활성화/비활성화할 수 있습니다.
   - 각 플러그인의 생명주기를 독립적으로 관리하여 앱의 안정성을 보장합니다.

2. **확장 포인트 (Extension Points)**
   - **명령어 팔레트**: 플러그인이 새로운 명령어를 등록하여 단축키로 실행할 수 있게 합니다.
   - **사이드바 뷰**: 사이드바에 독자적인 UI 탭을 추가하여 전용 도구를 제공할 수 있습니다.
   - **이벤트 구독**: 파일 열기, 저장 등의 이벤트를 감지하여 자동화된 작업을 수행할 수 있습니다.

## 📦 공식 지원 플러그인
### 1. 템플릿(Templates) 플러그인
사용자가 미리 정의한 마크다운 양식을 현재 문서에 즉시 삽입할 수 있는 기능을 제공합니다.
- **템플릿 저장소**: `.mark-explorer/templates/` 폴더 내의 `.md` 파일을 템플릿으로 인식합니다.
- **주요 기능**:
  - **단축키**: `Option + T` (Mac) / `Alt + T` (Windows)를 통한 템플릿 검색 및 삽입.
  - **변수 치환**: `{{date}}`, `{{time}}`, `{{title}}` 등 동적 데이터를 삽입 시 자동으로 변환합니다.
  - **커서 위치 삽입**: 현재 에디터의 커서가 위치한 곳에 정확히 마크다운 텍스트를 주입합니다.

## ⚠️ 제약 사항 및 알려진 이슈
- **단축키 안정성 (Mac OS)**: Mac OS의 시스템 자판 레이아웃 특성으로 인해 `Option + T` 단축키가 에디터 포커스 상황에서 일관되게 작동하지 않는 이슈가 보고되었습니다. 현재 이벤트 캡처링 방식을 통해 보완 중이며, 상세 보고서는 [여기](./errors/open-template---shortcut-error/issue-20260421.md)를 참조하십시오.
- **보안**: 플러그인은 정의된 `App` API를 통해서만 시스템 자원에 접근할 수 있습니다.

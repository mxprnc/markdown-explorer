# AI 통합 및 MCP 제품 명세 (ai-integration.md)

Mark Explorer는 최신 LLM(Gemini)과 MCP(Model Context Protocol)를 결합하여 지능적인 마크다운 편집 환경을 제공합니다.

## 🌟 핵심 가치
- **문맥 이해**: 작성 중인 문서의 맥락을 파악하여 관련 정보를 제안합니다.
- **외부 도구 확장 (MCP)**: AI가 단순한 텍스트 생성을 넘어 외부 데이터와 도구를 직접 사용할 수 있게 합니다.
- **개발자 친화적**: 사용자가 직접 MCP 서버를 연결하여 자신만의 AI 워크플로우를 구축할 수 있습니다.

## 🛠 주요 기능
1. **Gemini AI 어시스턴트**
   - 질문 답변, 텍스트 요약, 번역, 문법 교정 등의 기능을 에디터 내에서 바로 수행합니다.
   - **에이전트 기능**: 사용자가 허용한 폴더 범위 내에서 파일 생성, 수정, 리서치 보고서 작성 등의 작업을 자율적으로 수행합니다.
   - 플러그인 형태로 구현되어 필요할 때만 활성화하여 사용할 수 있습니다.

2. **MCP (Model Context Protocol) 연동**
   - **표준화된 확장**: AI 모델이 특정 서비스(GitHub, Slack 등)에 종속되지 않고, 표준화된 프로토콜을 통해 외부 도구와 데이터를 사용합니다.
   - **도구 호출(Function Calling)**: AI가 MCP 서버에서 제공하는 '도구(Tools)'를 인식하고 적재적소에 호출하여 실제 액션을 수행합니다.
   - **보안**: 사용자가 직접 제어하는 MCP 서버를 통해 민감한 데이터를 처리하고 AI에는 필요한 결과만 전달할 수 있습니다.

3. **AI 보안 감사 (개발 예정)**
   - Gemini를 활용하여 작성된 코드나 플러그인의 보안 취약점을 자동으로 분석합니다.

## 🏗 구현 아키텍처 및 전략
- **브레인 & 림(Brain & Limbs) 구조**: Gemini는 의사결정을 내리는 '뇌' 역할을 수행하고, MCP는 외부 환경과 상호작용하는 '팔과 다리' 역할을 수행합니다.
- **파일 시스템 접근 권한**:
  - 브라우저의 **File System Access API**를 통해 사용자가 명시적으로 허용한 폴더에 대해서만 에이전트 작업이 가능합니다.
  - 사용자가 '허용'을 클릭한 세션 내에서만 파일 쓰기 권한이 유효하며, 이는 AI의 오동작을 방지하는 보안 계층 역할을 합니다.
- **플랫폼별 MCP 대응**:
  - **Web 환경**: 브라우저 샌드박스 제약으로 인해 `SSE(Server-Sent Events)` 브릿지를 통한 외부 MCP 서버 연동을 지원합니다.
  - **Desktop 환경 (향후)**: `stdio` 기반의 로컬 프로세스 통신을 통해 OS 자원(로컬 쉘, 파일 시스템 전체 등)에 대한 더 깊은 통합을 지원할 수 있습니다.

## 🖥 사용자 인터페이스 (UI)
- **AI 채팅창**: 하단 또는 사이드바의 전용 인터페이스를 통해 AI와 대화합니다.
- **MCP 설정**: 설정 페이지에서 외부 MCP 서버의 URL을 등록하고 연결 상태를 확인합니다.

## ⚠️ 기술적 요구사항
- Gemini 기능을 사용하기 위해서는 개인 Google AI API 키가 필요합니다.
- 외부 MCP 서버와의 통신을 위해 로컬 네트워크 접근 권한(CORS 설정 포함)이 요구될 수 있습니다.
- 브라우저 환경에 따라 File System Access API 지원 여부가 다를 수 있습니다.

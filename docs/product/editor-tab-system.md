# 에디터 탭 시스템 (Editor Tab System)

본 문서는 Mark Explorer의 핵심 기능 중 하나인 VSCode 스타일의 탭 관리 시스템에 대한 제품 명세를 다룹니다.

---

## 1. 개요
사용자가 여러 파일을 동시에 효율적으로 탐색하고 편집할 수 있도록 VSCode와 유사한 탭 인터페이스를 제공합니다. "미리보기(Preview)"와 "영구 열기(Permanent)" 상태를 분리하여 과도한 탭 생성을 방지하고 정돈된 작업 환경을 유지합니다.

## 2. 주요 기능

### 2.1 탭 상태 구분
| 상태 | 시각적 특징 | 동작 원리 |
| :--- | :--- | :--- |
| **미리보기 (Preview)** | 제목이 *이탤릭체(Italic)*로 표시됨 | Explorer에서 파일을 **단일 클릭**했을 때 활성화됨. 다른 파일을 단일 클릭하면 기존 미리보기 탭이 교체됨. |
| **영구 열기 (Permanent)** | 제목이 일반 폰트로 표시됨 | Explorer에서 파일을 **더블 클릭**하거나, 미리보기 탭을 **더블 클릭**하여 고정(Pin)했을 때 활성화됨. |

### 2.2 핀 고정 (Pinning)
- 미리보기 상태의 파일은 자동으로 고정되지 않습니다.
- 다음 동작을 통해 영구 탭으로 전환할 수 있습니다:
    - 탭 제목 영역을 **마우스 더블 클릭**
    - 파일 내용을 수정(Edit)하기 시작할 때
    - 탭 우클릭 메뉴에서 **Pin** 선택

### 2.3 탭 컨텍스트 메뉴 (우클릭)
탭 제목을 우클릭했을 때 나타나는 핵심 액션들입니다:
- **Close**: 현재 선택된 탭을 닫습니다.
- **Pin**: 미리보기 탭을 영구 탭으로 고정합니다.
- **Close Others**: 현재 탭을 제외한 다른 모든 탭을 닫습니다.
- **Close All**: 모든 탭을 닫고 빈 화면으로 돌아갑니다.
- **Copy Relative Path**: 해당 파일의 상대 경로(`./path/to/file`)를 클립보드에 복사합니다.
- **Reveal in Finder**: (데스크탑 연동 시) OS의 파일 탐색기에서 해당 파일 위치를 엽니다.

## 3. 사용자 인터랙션 상세 (UX)

### 3.1 파일 탐색기(Explorer) 연동
- **단일 클릭**: 파일을 미리보기 모드로 엽니다. 이미 열려있는 미리보기 탭이 있다면 그 위치에서 내용만 바뀝니다.
- **더블 클릭**: 새로운 영구 탭을 생성하거나, 기존에 열려있던 탭을 활성화합니다.

### 3.2 탭 바(Tab Bar) 동작
- **액티브 상태**: 활성화된 탭 상단에는 테마의 프라이머리 컬러(Primary Color) 바가 나타나 시각적으로 강조됩니다.
- **손가락 포인터 (Cursor)**: 모든 클릭 가능한 요소(탭, 닫기 버튼) 위에 마우스를 가져가면 손가락 모양(`pointer`)으로 포인터가 변경되어 버튼임을 명확히 인지시킵니다.
- **가로 스크롤**: 탭이 많아질 경우 가로 스크롤을 통해 모든 탭에 접근할 수 있습니다.
- **드래그 앤 드롭 (Drag & Drop)**: 현재 탭의 드래그 상태 감지가 구현되어 있으며, 향후 탭 순서 변경 및 Pane 간 이동 기능이 추가될 예정입니다. (현재는 시각적으로 드래그만 가능)

### 3.3 컨텍스트 메뉴 상호작용
- **연속 우클릭**: 다른 탭을 우클릭할 경우 기존 메뉴가 자동으로 닫히고 새로운 메뉴가 즉시 표시됩니다. (전역 리스너 방식)
- **외부 영역 클릭**: 메뉴 외부를 클릭하거나 우클릭하면 메뉴가 닫힙니다.

## 4. 기술적 로직
- **TabUtils**: 탭의 추가/삭제/핀 고정/중복 체크 등의 비즈니스 로직은 순수 함수(Pure function)로 분리되어 유효성 검사가 완료되었으며, 성능 최적화를 위해 불필요한 리렌더링을 방지하고 있습니다.
- **Event Propagation & Global Listener**: 
    - `fixed` 포지셔닝을 사용하여 분할 창 모드에서도 위치가 어긋나지 않도록 설계되었습니다.
    - 차단용 레이아웃 없이 `window` 레벨에서 이벤트를 가로채어 닫으므로, 메뉴가 열린 상태에서도 다른 요소와의 상호작용이 원활합니다.

# 파일 탐색기 상호작용 (File Explorer Interaction)

본 문서는 Mark Explorer의 파일 탐색기(Explorer)에서 제공하는 사용자 상호작용 및 컨텍스트 메뉴 시스템에 대한 제품 명세를 다룹니다.

---

## 1. 개요
로컬 시스템의 파일 구조를 시각화하고, 직관적인 마우스 조작을 통해 파일 관리 및 탐색을 수행할 수 있는 기능을 제공합니다. VSCode의 고급 인터랙션을 모델링하여 부드럽고 일관된 UX를 지향합니다.

## 2. 주요 상호작용

### 2.1 마우스 클릭 동작
| 동작 | 대상 | 결과 |
| :--- | :--- | :--- |
| **단일 클릭** | 폴더 | 폴더의 펼침/접힘 상태를 토글함. |
| **단일 클릭** | 파일 | 파일을 **미리보기(Preview)** 모드로 에디터에 표시함. |
| **더블 클릭** | 파일 | 파일을 **영구 열기(Permanent)** 모드로 새 탭으로 여거나 활성화함. |

### 2.2 Hover 피드백
- 항목 위에 마우스를 가져가면 배경색이 하이라이트됩니다.
- 항목 우측에 퀵 액션 버튼(상대 경로 복사, 이름 변경 등)이 나타납니다.
- 클릭 가능한 모든 요소에는 `pointer` 커서가 적용되어 시각적 피드백을 제공합니다.

### 2.3 드래그 앤 드롭 (Web 전용)
- 파일을 드래그하여 상태를 전송할 수 있는 기반이 마련되어 있으며, 향후 에디터 영역이나 다른 탭바 영역으로의 이동 기능이 고도화될 예정입니다.

## 3. 컨텍스트 메뉴 (우클릭)

### 3.1 메뉴 항목 (항목별 차별화)
- **공통**:
    - **Rename**: 파일이나 폴더의 이름을 변경할 수 있는 모달을 띄웁니다.
    - **Delete**: 항목을 삭제합니다 (실제 파일 시스템 연동).
- **폴더(Directory) 전용**:
    - **New File**: 해당 폴더 하위에 새 파일을 생성합니다.
    - **New Folder**: 해당 폴더 하위에 새 폴더를 생성합니다.

### 3.2 메뉴 UX 상세
- **정확한 위치 지정**: `clientX/Y` 좌표와 `fixed` 포지셔닝을 결합하여, 스크롤 위치나 사이드바 크기에 영향받지 않고 정확히 마우스 클릭 지점에 메뉴가 나타납니다.
- **연속 우클릭 지원 (VSCode Style)**:
    - 메뉴가 이미 열려 있는 상태에서 다른 항목을 우클릭하면, 기존 메뉴가 즉시 닫히고 **새로운 항목에 대한 메뉴가 새 위치에 즉시 나타납니다.**
    - 이를 위해 차단형 배경 레이어(Blocking Backdrop) 대신 **전역 이벤트 리스너** 방식을 채택하여 끊김 없는 조작을 지원합니다.
- **외부 클릭 폐쇄**: 메뉴 외부의 빈 공간이나 다른 UI 요소를 클릭(또는 우클릭)하면 현재 열린 메뉴가 자동으로 닫힙니다.

### 3.3 모바일 전용 상호작용 (Mobile Interactions)
- **File Info Modal**: 파일 메타데이터 확인 및 이름 변경을 위한 모바일 최적화 전용 모달을 제공합니다.

## 4. 기술적 구현 특징
- **Event Propagation**: 각 컨텍스트 메뉴 트리거는 `e.stopPropagation()`을 사용하여 전역 닫기 리스너와의 간섭을 방지하고 정확한 타겟에 대해서만 메뉴를 렌더링합니다.
- **Theme Integration**: 다크 모드 및 라이트 모드 테마에 완벽히 대응하며, 시스템 폰트와의 통합을 통해 네이티브 앱과 같은 시각적 일치감을 제공합니다.

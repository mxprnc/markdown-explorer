# Markdown Explorer v1 - 개발 히스토리 및 트러블슈팅 정리

이 문서는 AI와 함께 `Markdown Explorer` 프로젝트를 진행하며 추가한 주요 기능, 발생했던 이슈, 그리고 해결 방법들을 기록한 기술 문서입니다.

---

## 1. 마크다운 스타일링 및 GitHub 스타일 적용
* **목표:** 개발자들에게 친숙한 GitHub Flavored Markdown(GFM) 스타일 테마 반영. (Ligth/Dark 모드 지원)
* **내용:** 
  * `CSS`를 활용해 `.tiptap-wrapper`, `.preview-wrapper` 등 렌더링 컨테이너에 세밀한 Typographic 스타일 세팅.
  * 테이블(Table), 인용구(Blockquote), 인라인 코드(`Code`), 리스트 패딩 등을 깔끔하게 정돈.

## 2. 유튜브 동영상 첨부 및 편의 기능 (Copy 버튼)
* **목표:** 일반적인 `https://www.youtube.com/watch?v=...` 텍스트 링크를 자동으로 파싱하여 에디터 및 Preview 화면 안에서 즉시 재생 가능한 임베드(Embed) 플레이어로 변환.
* **내용 & 해결방식:**
  * ProseMirror(TipTap)의 커스텀 Node 및 Extension(`YoutubeExtension`)을 제작하여, 마크다운 URL 패턴을 감시.
  * 유튜브 썸네일/플레이어 우측 하단에 **[링크 복사] 버튼**을 붙박이 UI로 구현.
* **이슈 & 트러블슈팅:**
  * **에러:** `expo-clipboard` 모듈을 통한 클립보드 복사 시, 데스크탑 웹 브라우저 환경에서 경고창(Alert)이 네이티브 앱과 달라 발생.
  * **해결:** `Platform.OS === 'web'` 분기 처리를 도입하고, 복사 버튼 클릭 후 `e.preventDefault()`, `e.stopPropagation()` 방어 코드를 통해 편집기의 텍스트 선택(Text Selection) 포커스 충돌 방지.

## 3. 로컬 파일 시스템 수정내역 실시간 저장
* **목표:** 에디터에서 파일을 수정한 내용을 즉시 사용자의 로컬 드라이브 속 원래 파일에 안전하게 덮어쓰기. (`Cmd+S` 동기화)
* **내용 & 해결방식:**
  * HTML5 **File System Access API (`showDirectoryPicker`)** 를 활용해 브라우저 보안 규격을 만족하며 폴더의 권한을 획득(+폴더 내 파일 리스팅).
  * `index.tsx`에서 파일 객체를 `createWritable()`로 열고 덮어쓰는(Save) 로직(`handleSaveToDisk`) 구현.
* **이슈 & 트러블슈팅:**
  * **이슈:** 파일을 다른 파일로 이동했다가 다시 열면 이전에 편집한 최신 내역이 증발하는 현상 (메모리 상태가 디스크 상태를 미반영).
  * **해결:** 에디터 모드에서 `Cmd+S`(또는 상단 저장 버튼 클릭) 시, TipTap의 최신 마크다운 Content를 불러와 `app/index.tsx` 부모 컴포넌트의 상태를 동시에 갱신하고, 물리적인 디스크 쓰기를 함께 대기(`await`) 후 완료 알림을 띄우는 완전 동기화 체계로 개편.

## 4. 실시간 TOC (목차 네비게이션) 기능 추가
* **목표:** 문서 작성 중 우측 패널에 자동으로 H1 ~ H6 제목들을 추출하여 네비게이션 목차로 시각화.
* **내용 & 해결방식:**
  * 마크다운 텍스트를 실시간으로 파싱(`split('\n')` 후 정규식 `^(#{1,6})\s+`)하여 헤딩 리스트 추출.
  * TOC 항목 클릭 시 **부드러운 스크롤 이동**(`scrollIntoView({ behavior: 'smooth' })`).

## 5. 인라인 이미지 파일명 변경 (Rename) + 너비 자동 크기(Scale) 조절
* **목표:** Markdown의 인라인 이미지 태그 속성을 UI 버튼만으로 편하게 조작. 원본 파일명을 디스크에서 변경하고 마크다운 파싱값까지 동기화, 비율(Scale) 커스터마이징.
* **내용 & 해결방식:**
  * 이미지가 렌더되는 `NodeView` 하단에 가상의 `div`를 붙여 **Scale 입력칸, Rename 버튼** UI를 주입.
  * Rename 완료 후, `File System Access API` 로컬 파일의 이진 데이터 복제본(Blob) 생성 -> 새 이름 쓰기 -> 기존 파일 삭제(`removeEntry`).
  * Width(Scale) 조절 시 "숫자만 덜컥 쳐도 자동으로 `%`를 붙이는 오토-완성(Smart Fallback)" 구현.
* **이슈 & 트러블슈팅:**
  * **에러:** Scale 수치를 입력하고 `Enter`를 눌러 DOM(NodeView)이 교체/파괴(destroy)되는 시점에, 포커스 분실(blur) 이벤트가 연달아 터지며 언마운트된 노드 영역에 접근하다 ProseMirror 내부 에러(crash) 발생.
  * **해결:** `isDestroyed = false;` 안전 장치(Flag) 설치. DOM 소멸 직전 `destroy` 콜백에서 플래그를 올려, 무효화된 노드 쪽으로는 어떠한 업데이트나 명령도 들어가지 않도록 이벤트 핸들러 라이프사이클 무력화(Patch).

## 6. 사이즈 리사이징(Drag & Drop Pane Resizer)
* **목표:** 데스크탑 IDE(VS Code 등)처럼 사용자가 마우스 드래그를 통해 좌/우 사이드바와 메인 패널의 넓이를 자유자재로 설정.
* **내용 & 해결방식:**
  * React Native `PanResponder` API를 활용, Explorer 좌측 뷰, Middle 파일 리스트 뷰, 에디터 TOC 우측 뷰 각각에 드래그 센서(Divider Line) 배치.
* **이슈 & 트러블슈팅:**
  * **에러:** 브라우저 환경에서 경계선을 마우스로 빠르게 휙 드래그하다 보면, 배경화면의 텍스트가 파랗게 지정되는 '커서 Text Select' 현상이 발생하며 드래그 이벤트가 강제로 멈춰버림.
  * **해결:** 사용자가 터치/클릭을 잡은 순간(`onPanResponderGrant`) React State `isResizing`을 `true`로 켜서 앱 전체의 CSS 포인터 상태를 `user-select: none`으로 락킹(잠금). 마우스를 뗄 때(`Release/Terminate`) 다시 원복시켜 원천적인 마우스 간섭 회피.

## 7. 다중 파일 멀티 탭(Multi-Tabs) 에디팅
* **목표:** VS Code 등 전문 에디터처럼 여러 마크다운 파일을 동시에 열어두고 탭 단위로 빠르게 스위칭하며 편집할 수 있도록 환경 구축.
* **내용 & 해결방식:**
  * 상위 상태(`openedFiles` String 배열 형식)를 정의하여 현재 열려 있는 탭 목록 관리.
  * Preview(Files 탭) 모드, WYSIWYG Editor 탭 모드 양쪽 상단 컴포넌트에 가로 스크롤이 가능한 네비게이션 Tab Bar(`renderTabBar`)를 마운트.
  * 파일 탭의 닫기 버튼(✕) 클릭 시 `openedFiles` 배열에서 제거하고, 활성화된 현재 탭을 지우는 경우 가장 우측(마지막) 탭으로 똑똑하게 포커스를 폴백(Fallback).
* **이슈 & 주요특징:**
  * 로컬 메모리 백업 동기화: 새로운 파일로 탭을 갈아탈 때마다, 기존 탭의 에디터 값(`editorContent`)을 `setLocalFiles(prev => ({ ...prev, [selectedFile]: editorContent }))`로 메모리 상에 임시 백업해둠. 
  * 이 덕분에 물리적인 로컬 디스크 저장을 수동으로 누르지 않더라도 탭 사이를 스위칭할 때 작성 중이던 내용이 유실되지 않고 안전하게 일시 보존됨.

## 8. 스플릿 뷰(Split View) 화면 분할 모드
* **목표:** VS Code처럼 화면을 좌우로 분할하여 서로 다른 문서를 동시에 편집하거나 참조할 수 있는 환경 구축.
* **내용 & 해결방식:**
  * `isSplitMode`, `activePane` 상태를 도입하여 두 개의 독립된 에디터 인스턴스 레이아웃 구현.
  * `targetPane` 인자를 추가하여 `handleSelectFile`, `closeTab`, `handleSaveToDisk` 로직이 활성화된 창(Pane)에만 독립적으로 동작하도록 리팩토링.
  * 화면 상단 분할 모드 토글 버튼 추가 및 분할 시 현재 열린 문서 내용을 우측 창으로 복제하여 초기화.

## 9. 탭 드래그 앤 드롭(Drag & Drop) 및 화면 분할 트리거
* **목표:** 탭을 마우스로 잡아 끌어서 다른 창으로 옮기거나, 우측 빈 공간에 드롭하여 즉시 화면을 분할하는 직관적인 UX 구현.
* **내용 & 해결방식:**
  * HTML5 **Drag and Drop API**를 활용, 탭 영역(`div draggable="true"`) 및 에디터 각 창에 `onDrop` 핸들러 배치.
  * 탭을 드래그하여 에디터 우측 절반('Split Zone')에 드롭하면 자동으로 `isSplitMode`를 활성화하며 해당 파일을 우측 창으로 이동.
* **이슈 & 트러블슈팅:**
  * **이슈 1:** 드래그 시작 시 탭 안의 파일명이 텍스트 블록 지정(Highlight)되면서 브라우저가 드래그를 취소함 -> `userSelect: 'none'`, `selectable={false}` 3중 차단으로 해결.
  * **이슈 2:** 드롭 완료 후에도 보이지 않는 투명 레이어(onDrop 감지용 div)가 `zIndex` 상위에 남아 에디터 클릭 및 타이핑을 방해함 -> 드롭 직후 `setDraggingTab(null)` 상태 전이를 통해 레이어를 즉시 파괴(Cleanup)하도록 수정.
  * **이슈 3:** 브라우저(크롬 등)에서 `DataTransfer.setData`가 명시되지 않으면 보안상 드래그를 무효화함 -> `dt.setData('text/plain', file)`를 의무화하여 웹 호환성 확보.

## 10. 스플릿 뷰 지능형 TOC (Active Pane Sync)
* **목표:** 화면이 분할된 상태에서도 현재 사용자가 편집 중인 창의 내용에 맞춰 목차가 실시간으로 업데이트되도록 동기화.
* **내용 & 해결방식:**
  * TOC 추출 로직(`tocList` useMemo)에서 `activePane`을 감시하도록 설정.
  * 사용자가 클릭(Focus)한 Pane 번호에 따라 `editorContent`(Pane 1)와 `editorContent2`(Pane 2) 중 적절한 데이터를 선택하여 목차를 재계산.
  * 이를 통해 서로 다른 파일을 양쪽에 열어두고 창을 옮겨 다닐 때마다 해당 문서의 목차로 자연스럽게 전환됨.

## 11. 계층형 탐색기(Hierarchical Explorer) 및 트리 UI
* **목표:** 프로젝트 루트 폴더뿐만 아니라 모든 하위 폴더와 파일(문서, 이미지 등)을 VS Code 스타일의 계층 구조로 탐색.
* **내용 & 해결방식:**
  * **재귀적 스캔:** `File System Access API`를 통해 폴더 내 모든 항목을 재귀적으로 탐색하여 트리 구조의 `fileSystemData` 상태 구축.
  * **트리 UI 구현:** 폴더 깊이(Depth)에 따른 들여쓰기와 아이콘(📁, 📄, 🖼️)을 적용한 `renderFileSystem` 함수 구현.
  * **폴더 접기/펼치기:** `expandedFolders` 상태를 통해 개별 폴더를 토글할 수 있는 UX 제공. (초기 로딩 시 모두 접힘 상태)
  * **이중 탐색 구조:** 왼쪽 "Explorer"에서는 전체 트리를 보여주고, 가운데 "Markdown Explorer" 창에서는 현재 선택된 폴더 내부의 직계 자식들만 보여주는 IDE식 이중 네비게이션 체계 도입.

## 12. 성능 최적화: 레이지 로딩(Lazy Loading) 시스템
* **목표:** 수천 개의 파일이 있는 대규모 프로젝트 폴더를 열더라도 로딩 지연 없이 즉시 사용 가능하도록 최적화.
* **내용 & 해결방식:**
  * **On-Demand Directory Scanning:** 초기 로딩 시 모든 하위 폴더를 한꺼번에 읽지 않고, 사용자가 폴더를 펼치거나 진입할 때만 해당 디렉터리를 스캔하도록 변경.
  * **Lazy File Content Reading:** 수많은 텍스트 파일의 내용을 처음부터 메모리에 올리는 대신, 에디터에서 실제로 파일을 클릭하여 **열람하는 순간**에만 디스크로부터 `text()`를 읽어오도록 개선.
  * **캐싱 로직:** 한 번 읽어온 파일 내용은 `localFiles` 상태에 보관하여, 재열람 시 디스크 접근 없이 즉각적으로 화면에 표시.
* **성능 결과:** 초기 스캔 대상이 수천 개에서 수십 개(루트 항목)로 줄어들어 대규모 프로젝트 환경에서도 끊김 없는 탐색 환경(Zero-Lag) 달성.

## 13. 이미지 통합 뷰어 및 인터랙티브 탐색 기능 (Zoom & Pan)
* **목표:** 에디터 내에서 이미지 파일을 별도의 툴 없이 즉시 열람하고, 확대/축소 및 드래그 이동을 통해 세부 내용을 확인할 수 있는 프리미엄 뷰어 환경 구축.
* **내용 & 해결방식:**
  * **통합 이미지 렌더링:** `handleSelectFile` 로직을 확장하여 이미지 파일 선택 시 `URL.createObjectURL`을 통해 Blob URL을 생성하고 전용 `ImageViewer` 컴포넌트로 연결.
  * **인터랙티브 줌(Zoom):** `useState`를 활용한 배율 제어 로직을 구현하여 10%~1000%까지 자유로운 확대/축소 지원.
  * **부드러운 드래그(Panning):** 마우스 이벤트 핸들러(`onMouseDown`, `onMouseMove` 등)를 통해 확대된 이미지의 특정 위치로 시야를 옮기는 직관적인 드래그-앤-판 기능 구현.
  * **상태 동기화:** 툴바를 통해 배율과 위치를 즉시 초기화하는 Reset 기능 및 다중 창(Split View) 동시 이미지 열람 지원.

## 14. 하이브리드 파일 관리 시스템 (Context Menu & Hover Actions)
* **목표:** 앱 내부에서 파일/디렉터리의 생성, 삭제, 이름 변경을 직관적이고 빠르게 수행할 수 있는 전문 IDE 수준의 관리 환경 구축.
* **내용 & 해결방식:**
  * **하이브리드 UX 설계:** 
    * **Context Menu:** 우클릭 시 나타나는 전용 메뉴를 통해 고급 기능(삭제, 이름 변경) 제공.
    * **Hover Actions:** 가운데 리스트 항목에 마우스를 올릴 때만 나타나는 인라인 버튼(Pencil, Trash)으로 빠른 접근성 확보.
  * **파일 시스템 변이 처리:**
    * **Rename:** `FileSystemHandle.move()` API를 활용하여 디렉터리와 파일 이름을 물리적으로 변경. 이름 변경 시 해당 파일을 열고 있는 모든 탭의 경로를 실시간으로 업데이트하여 작업 연속성 유지.
    * **Delete:** `removeEntry(recursive: true)`를 통해 선택한 항목을 삭제하고, 관련된 모든 로컬 상태(Tabs, Selections, Cache)를 즉시 청소.
  * **UI/UX 디테일:** 
    * 이름 변경을 위한 세련된 모달(Modal) UI 구현.
    * `StyleSheet.absoluteFill`을 활용한 컨텍스트 메뉴 바깥 클릭 감지 및 자동 닫힘 로직 적용.

  * **글로벌 최신 에이전트 라인업 통합:**
    * **Google Gemini**: 기본 추천 모델인 `gemini-3.5-flash` 및 추론 및 고부하 개발에 특화된 `gemini-3.1-pro-preview`를 적용하고, 통신 방식을 `v1beta` 프로토콜로 표준화.

  * **단축키 최종 해결 (Cmd + Enter):**
    * 기본 브라우저의 전송 및 개행(줄바꿈) 충돌을 말끔히 해결하고 IDE 수준의 사용성을 만들기 위해, `onKeyPress`에서 `metaKey/ctrlKey` 조합을 가로채서(Intercept) 매끄럽게 채팅이 전송되도록 입력 UX 최적화.
  * **지능형 제공자별 자가 복구(Self-Healing):**
    * 로컬 스토리지에 옛 모델명이나 어긋난 제공자 캐시가 남아 404/403 통신 오류를 일으키는 문제를 원천 차단하기 위해, 앱 시작 시 저장 상태의 무결성을 점검하여 타겟 제공자의 현재 최신 권장 모델로 자동 회귀(Fallback)시키는 스마트 복구 로직 도입.
* **이슈 & 트러블슈팅:**
  * **에러 (404 Not Found):** 모델 ID 명칭(`gemini-3.1-pro`)이 프리뷰 단계에서 요구하는 정확한 형식(`-preview`)과 맞지 않아 발생.
  * **해결:** 구글 API 문서 조사를 통해 정확한 모델 ID 사양을 파악하고, 계정 권한에 따른 호환성을 위해 `latest` 및 `preview` 별칭(Alias)을 모두 지원하도록 리팩토링.
  * **에러 (sticky settings):** 코드를 고쳐도 사용자의 브라우저 캐시에 옛날 모델 정보가 남아있어 계속 404가 뜨는 현상.
  * **해결:** `useEffect` 초기화 단계에서 `availableModels` 목록과 대조하여 유효성을 검사하는 코드를 추가하여 원천 차단.

## 21. 컨텍스트 기반 이미지 경로 해결 (Context-aware Image Resolution)
* **목표:** 프로젝트 루트 폴더가 아닌, 깊은 하위 폴더에 위치한 마크다운 문서에서도 상대 경로(`./`, `../`)로 지정된 이미지가 정상적으로 표시되도록 개선.
* **내용 & 해결방식:**
  * **현재 파일 경로 연동:** `resolveImage` 함수가 현재 열려 있는 문서의 경로 정보를 인자로 받아, 해당 파일이 속한 디렉터리를 기준으로 이미지를 찾도록 로직 변경.
  * **경로 정규화(Stack-based Pathing):** `../`를 통해 상위 폴더로 이동하는 경로를 스택(Stack) 방식으로 처리하여 최종 물리적 위치를 정확히 계산.
  * **공통 적용:** 에디터 인스턴스뿐만 아니라 마크다운 미리보기(`Preview`) 컴포넌트에도 동일한 이미지 해결 로직을 주입하여 데이터 일관성 확보.

## 22. 헤더 경로 복사 대상 고도화 (Root Path Targeting)
* **목표:** 프로젝트 전체의 기준점이 되는 루트 폴더의 경로를 빠르게 획득할 수 있도록 상단 버튼의 동작을 개선.
* **내용 & 해결방식:**
  * **동작 변경:** 상단 로고 옆에 표시된 경로 우측의 `@` 아이콘과 복사 아이콘이 현재 파일이 아닌 **'Markdown Explorer로 열어둔 루트 폴더'**를 대상으로 동작하도록 수정.
  * **기대 효과:** 현재 편집 중인 파일의 경로는 파일 목록/트리의 호버 버튼으로 얻고, 상단 버튼으로는 프로젝트 전체의 절대/상대 경로(Root)를 간편하게 복사할 수 있는 이중 구조 확립.

## 26. 지능형 스플릿 뷰 TOC 네비게이션 고도화
* **목표:** 화면 분할 유무에 관계없이 TOC 목차 클릭 시 현재 활성화된 창(Pane)의 해당 섹션으로 정확하고 부드럽게 스크롤 이동.
* **내용 & 해결방식:**
  * **Pane 기반 DOM 매칭:** 각 창에 고유 ID(`pane-1`, `pane-2`)를 부여하고, `handleTOCClick` 시 현재 활성 창 내부에서만 헤더를 검색하도록 스코프 제한.
  * **좌표 계산 최적화:** `getBoundingClientRect()`를 활용해 뷰포트 상대 좌표를 계산하고, 상단 헤더에 가려지지 않도록 **스크롤 오프셋(Offset)** 보정 로직 적용.
  * **강건한 헤더 매칭:** 헤더의 텍스트뿐만 아니라 문서 내 인덱스(Index) 정보를 함께 활용하여 동일한 제목이 여러 개일 경우에도 정확한 위치 탐색.

## 27. 익스플로러(Files) 모드 화면 분할 및 독립 제어
* **목표:** 에디터뿐만 아니라 파일 미리보기(Files) 모드에서도 두 개의 문서를 동시에 비교하며 읽을 수 있는 멀티 뷰 환경 구축.
* **내용 & 해결방식:**
  * **독립적 상태 관리:** 미리보기 창마다 별도의 열린 파일 목록(`openedFiles2`)과 선택된 파일(`selectedFile2`) 상태를 부여하여 독립적인 문서 서핑 지원.
  * **포커싱 자동화:** 탐색기에서 파일을 클릭하면 마지막으로 터치/클릭하여 **활성화된(Active) 창**에서 해당 파일이 열리도록 지능형 라우팅 구현.
  * **TOC 연동:** 미리보기 모드에서도 현재 보고 있는 창의 내용에 따라 우측 목차가 실시간 동기화되도록 연동.

## 31. Cmd+P 로 파일 검색 (Quick Open File Search)
* **목표:** VS Code 등 전문 개발 도구처럼 `Cmd+P` 단축키를 활용하여 프로젝트 내 수많은 마크다운 및 이미지 파일을 키워드로 빠르게 탐색하고 즉시 열 수 있는 고속 탐색 환경 구축.
* **내용 & 해결방식:**
  * **전역 키 바인딩:** 앱 전역에 키보드 이벤트를 구독하여 `Cmd+P`(Windows/Linux 환경에서는 `Ctrl+P`) 입력 시 동작할 퀵 검색 창(`QuickOpenModal`)을 구현.
  * **지능형 검색 알고리즘:** 계층형 탐색기에서 재귀적으로 로드된 전체 파일 리스트를 평탄화(Flatten)하고, 초성/한글/영어 대소문자를 구분하지 않는 부분 일치(Sub-string match) 및 접두사 일치(Prefix match)에 우선적인 가중치를 부여하는 검색 필터 설계.
  * **키보드 중심 UX:** 검색 결과 창에서 마우스를 쓰지 않고 키보드 방향키(`ArrowUp`, `ArrowDown`)와 `Enter` 키로 타겟 파일을 확정하고 즉시 에디터나 미리보기(Files) 창에 로드할 수 있도록 완전한 단축키 인터랙션 지원.
* **이슈 & 트러블슈팅:**
  * **이슈 1 (브라우저 기본 기능 충돌):** 웹 브라우저 환경에서 `Cmd+P`를 누르면 브라우저 자체의 '페이지 인쇄(Print)' 팝업이 노출되면서 파일 검색 레이아웃을 가리고 렌더링에 간섭하는 문제 발생.
  * **해결 1:** 단축키 핸들러 맨 앞단에 `e.preventDefault()` 및 `e.stopPropagation()`을 실행하여 브라우저 고유 인쇄 이벤트를 원천 무력화.
  * **이슈 2 (포커스 유실):** 검색창을 닫거나 파일을 선택하고 나서 에디터(TipTap)의 기존 타이핑 포커스가 유실되어, 다시 에디터를 마우스로 직접 클릭해야 타이핑이 가능해지는 흐름 중단 현상.
  * **해결 2:** 모달 언마운트 라이프사이클에 맞춰 현재 활성화된 에디터 패널의 인스턴스를 찾아 강제로 `.focus()`를 주입함으로써 끊김 없는 편집 연속성 확보.

## 32. 마크다운 수식 렌더링 지원 (LaTeX Math Formula Rendering)
* **목표:** 마크다운 에디터와 미리보기(Preview) 화면에서 인라인 수식(`$...$`) 및 블록 수식(`$$...$$`)의 LaTeX 문법을 수학 공식 전용의 아름다운 벡터 그래픽으로 렌더링하여 공학 및 과학 문서 작성 경험 극대화.
* **내용 & 해결방식:**
  * **정교한 수식 파서 탑재:** `KaTeX` 엔진을 연동하여 마크다운 파서 및 뷰어 렌더링 파이프라인에서 수식 구분자(`$`, `$$`)를 고속으로 추출하고 이를 특수 수식 노드로 치환하는 렌더러 구현.
  * **디스플레이 모드 차별화:** 인라인 수식은 텍스트 줄바꿈을 해치지 않고 흐름 속에 조화롭게 안착시키고, 블록 수식은 중앙 정렬 및 세로 마진과 적절한 폰트 스케일업을 적용하여 문서 가독성 향상.
  * **실시간 동기화:** 에디터에서 타이핑하여 수식을 수정하는 즉시 미리보기 패널에서 딜레이 없이 실시간으로 변환 결과가 갱신되도록 상태 관리 연동.
* **이슈 & 트러블슈팅:**
  * **이슈 (웹 폰트 리소스 로드 에러):** Metro 번들러가 KaTeX CSS 내부에 선언된 `.woff2`, `.ttf` 등의 로컬 폰트 파일을 빌드하는 과정에서 웹 보안 규격 및 경로 참조 오류(`Importing local resources`)로 인해 빌드가 중단되거나 수식 폰트가 사각형( 깨짐)으로 표시되는 에러.
  * **해결:** 로컬 CSS 번들링 대신 공식 CDN 파일 서빙 방식으로 선회하여 `app/+html.tsx` 상단 템플릿에 공식 배포된 KaTeX 스타일시트를 전역 임포트함으로써, 번들링 부담을 없애고 모든 브라우저에서 공식 폰트가 깨지지 않고 깔끔하게 렌더되도록 최종 해결.

* **목표:** 단순히 하나의 인공지능 모델에 한정되지 않고, 다양한 글로벌 AI 제공자(OpenAI, Claude, Gemini)의 최신/추론 모델을 하나의 고풍스러운 대화창에서 번갈아 선택하며 실시간 문서 요약, 코딩 조언, 질문 답변을 진행할 수 있는 최첨단 AI 어시스턴트 허브 구축.
* **내용 & 해결방식:**
  * **2026년형 멀티 프로바이더 확장:** 
    * **Google Gemini:** `Gemini 3.5 Pro`, `Gemini 3.1 Pro (Reasoning)`, `Gemini 3.1 Flash` 등 추론 및 고속 처리를 아우르는 최신 라인업 대응.

    * **Anthropic Claude:** 가장 강력한 성능의 `claude-opus-4-6`, `claude-sonnet-4-6` 및 빠르고 경제적인 `claude-sonnet-4-5`, `claude-haiku-4-5` 지원.

    * 각 모델 제공자별 개별 API Key 입력을 하나의 **지능형 다목적 API Key 단일 필드**로 결합하여 사용자가 현재 선택한 어시스턴트에 맞춰 키를 보관/전송할 수 있도록 편리함 극대화.
  * **드래그 높이 조절 및 아카이브 시스템:**
    * 채팅 패널 상단 경계를 마우스로 드래그하여 채팅 영역의 높이를 확장할 수 있는 `PanResponder` 기반 수직 리사이저 장착.
    * AI와의 대화 결과를 마크다운 파일로 영구 보관할 수 있도록 "Archive" 기능을 고도화하여, 현재 탐색기 내 파일명을 추천해주는 실시간 오토플리트(Auto-complete) 지원.
* **이슈 & 트러블슈팅:**
  * **이슈 1 (로컬 스토리지 정보 고착으로 인한 404 에러):** 구글 API 명세 변경이나 구버전 모델 캐시가 브라우저에 남아있어 신규 버전 실행 시 통신 오류(404 Not Found)를 유발하는 현상.
  * **해결 1 (자가 복구 - Self-Healing):** `SettingsContext` 기동 단계에서 로컬 스토리지에 유보된 모델명이 현재 지원 모델 목록에 존재하지 않을 경우, 기본 안전 권장 모델(`gemini-2.5-pro` 등)로 강제 회귀(Fallback)시키는 자동 롤백 루틴을 내장하여 오작동 차단.
  * **이슈 2 (Playwright E2E UI 테스트 충돌):** 통합 설정 모달 UI의 레이아웃 개편 및 버튼 텍스트의 중복으로 인해 Playwright E2E 자동 테스트 시 Selector가 모호해져 테스트가 실패하는 현상.
  * **해결 2:** 모달 주요 요소에 고유한 `testID`와 접근성 식별값을 심어 테스트 격리성을 확보하고, API 키 필드가 단일화됨에 따라 테스트 명세 시나리오 역시 단일 dynamic-key 입력 흐름으로 통일하여 최종 CI/CD 패스 달성.

## 34. 테마 플러그인 시스템(Theme Plugin System) 및 VSCode 테마 임포터
* **목표:** 개발자 및 사용자의 취향에 맞는 맞춤형 문서 작성 환경을 제공하기 위해, 마크다운 에디터와 앱 인터페이스 전반의 색상 테마를 동적으로 변경할 수 있는 플러그인 아키텍처 및 외부 테마 가져오기 시스템 구축.
* **내용 & 해결방식:**
  * **테마 생명주기 및 API 구축 (`Manifest.ts`, `App.ts`):**
    * 기존의 `PluginManager` 아키텍처에 긴밀히 통합하여 테마를 플러그인의 한 분류(`type: 'theme'`)로 설계 및 표준 Manifest 규격화.
    * 플러그인이 런타임에서 테마를 핸들링할 수 있도록 `ThemeAPI`(`registerTheme`, `unregisterTheme`, `setActiveTheme`) 제공 및 EventBus 설계.
    * 웰컴 PoC용 다크 네온 테마인 `Tokyo Night` 및 `One Dark Pro` 테마를 내장 플러그인 형태로 구현하여 기본 탑재.
  * **VSCode 테마 호환 파서 개발 (`VSCodeThemeParser.ts`):**
    * VS Code용 JSON 테마 포맷을 직접 분석하고 정제하여 당사 규격의 HSL/HEX 색상 토큰으로 똑똑하게 번역해 주는 독립 파서 엔진 탑재.
    * JSON 텍스트 내에 섞여 있는 주석(한 줄 주석 `//`, 다중 줄 주석 `/* */`)을 완벽하게 필터링하여 오동작 없는 JSON 파싱 구현 및 색상 누락 시 라이트/다크 모드별 안전 폴백(Fallback) 보정 로직 내장.
* **이슈 & 트러블슈팅:**
  * **이슈 (Server Side Rendering localStorage ReferenceError):** Playwright E2E 웹서버(Node 환경) 사전 컴파일 단계에서, `useState` 내에 동기식으로 배치된 `localStorage.getItem()`을 호출하여 `localStorage is not defined` 에러로 전체 마운트가 붕괴되는 현상.
  * **해결:** `Platform.OS === 'web'` 체크 조건식 뒤에 `&& typeof localStorage !== 'undefined'` 안전 장치를 3중 추가하여 Node SSR 단계에서는 기본 폴백값으로 안전하게 넘어가고 클라이언트 브라우저에서만 정상 로드 및 쓰기 보존이 처리되도록 생명주기 정렬 패치.

    * 좌측 사이드바에 미려한 탭 레이아웃을 제공하고 4대 탭(**AI Assistant**, **Appearance**, **API Keys**, **Integrations**)으로 데이터 스코프 분류.
  * **정밀 폰트 스케일러 및 LocalStorage 영속화:**
    * **Appearance** 탭 내에 사용자의 접근성을 돕기 위해 **UI Font Size** 및 **Editor Font Size**를 1px 단위로 조절할 수 있는 정밀 스케일러 조작 버튼 구현.
    * 폰트 사이즈와 테마 모드 설정을 `localStorage`에 영구 보존하여 페이지 새로고침 시에도 사용자의 레이아웃 크기 배율과 디자인 테마가 완전히 보존되도록 구현.
    * UI Font Size 조절 시, 좌측 사이드바, 상태 바(Footer), AI Assistant 패널 타이틀, 설정 모달 내 요소들이 조화로운 비율로 연쇄 스케일링되도록 유동적 위계 스타일 설계.

## 36. 독립형 플러그인 & 테마 관리 사이드바(Dedicated Plugins Sidebar)
* **목표:** 설정(Settings) 모달 창 깊숙한 곳에 숨겨져 있던 플러그인 목록, 활성화 토글, 테마 스위칭, 임포터 폼 등의 옵션들을 메인 화면 좌측 액티비티 바(Activity Bar) 전용 일등 메뉴로 분리 개편하여, VS Code 및 Obsidian 수준의 편리한 확장 관리자 UX 구현.
* **내용 & 해결방식:**
  * **사이드바 확장 센터 구현 (`PluginsSidebar.tsx`):**
    * 액티비티 바에 퍼즐 조각 아이콘(`extension-puzzle-outline`)을 새롭게 장착하고, 클릭 시 탐색기 창 옆에 즉시 활성화되는 전용 플러그인 사이드바 설계.
    * **세그먼트 탭 필터 (ALL / PLUGINS / THEMES):** 사용자가 원하는 종류의 카드들만 모아볼 수 있는 고성능 탭 필터링 지원.
    * **실시간 탐색 검색 바:** 플러그인의 명칭과 상세 설명 텍스트를 실시간 완전 검색(FTS)하여 지연 없이 매치 항목을 렌더링하는 검색 인풋 바 제공.
    * **원클릭 테마 핫스왑 및 테마 임포터 내장:**
      * 테마 카드 하단의 `Apply Theme` 버튼 클릭 시 에디터와 컴포넌트 전반의 테마가 딜레이 없이 즉시 즉각 변경되는 핫스왑 지원.
      * 가져온 외부 커스텀 테마를 원클릭 삭제할 수 있는 휴지통 아이콘 탑재.
      * `ALL` 및 `THEMES` 탭 하단에 접이식 아코디언 컴포넌트로 테마 임포터를 배치하여 깔끔한 공간 효율성을 추구하고, `PLUGINS` 기능형 목록에서는 개념 혼선을 막기 위해 이를 자동 은닉하는 조건부 렌더링 도입.

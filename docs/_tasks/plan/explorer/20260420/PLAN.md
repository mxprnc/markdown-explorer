# Explorer 컨텍스트 메뉴 위치 수정 계획 (PLAN.md)

탐색기 컨텍스트 메뉴 위치 오류를 해결하기 위한 단계별 구현 계획입니다.

## 📅 실행 단계별 마인스톤

### Phase 1: 좌표 시스템 및 스타일 분석
- [x] `ContextMenu.tsx`의 현재 스타일(`position`) 및 좌표 적용 방식 분석.
- [x] `FileExplorer.tsx`에서 전달하는 `clientX`, `clientY` 값이 실제 뷰포트 대비 정확한지 재검증.

### Phase 2: 레이아웃 보정 및 고정 포지션 적용
- [x] `ContextMenu` 컴포넌트의 스타일을 `position: 'fixed'`로 변경하여 부모 컨테이너 오프셋 무시.
- [x] 블로킹 배경 레이어 제거 및 전역 `window` 리스너(`click`, `contextmenu`) 도입.
- [x] 각 메뉴 트리거 시 `e.stopPropagation()`을 추가하여 전역 리스너와의 간섭 방지.
- [x] 탭 컨텍스트 메뉴와 탐색기 컨텍스트 메뉴의 UX 일관성 확보.

### Phase 3: 최종 검증 및 예외 처리
- [x] 탐색기 내 깊은 계층의 폴더에서도 우클릭 위치가 정확한지 테스트.
- [x] 서로 다른 해상도 및 브라우저 창 크기에서 메뉴 위치 불일치 여부 확인.

## ⚠️ 기술적 고려사항
- `react-native`의 `View`는 웹에서 기본적으로 `position: relative` 성향을 띠므로, 이를 덮어쓰기 위해 `fixed` 스타일을 명시적으로 적용해야 함.
- 탭 컨텍스트 메뉴에 적용했던 성공 사례를 탐색기 메뉴에도 동일하게 수평 전개함.

# Preview Component E2E Test Goals

이 문서는 마크다운 미리보기(`Preview.web.tsx`) 및 이미지 뷰어(`ImageViewer.tsx`) 컴포넌트에 대한 E2E 테스트 최종 목표를 정의합니다.

## 1. 핵심 검증 목표

### A. 마크다운 미리보기 (Markdown Preview)
- [x] **표준 렌더링**: 텍스트 스타일(Bold, Italic), 리스트, 링크, 인용구 등 표준 마크다운 문법이 올바르게 HTML로 변환되는지 확인. (완료)
- [x] **고급 컴포넌트**: 수식(KaTeX), 표(Table), 머메이드(Mermaid) 다이어그램이 브라우저에서 시각적으로 렌더링되는지 확인. (완료)
- [x] **미디어 통합**: YouTube 링크가 iframe 플레이어로 자동 변환되는지, 이미지가 경로 해설을 통해 정상 노출되는지 확인. (완료)
- [x] **네비게이션 동기화**: 외부(TOC 등)에서의 스크롤 요청 시 해당 위치로 부드럽게 이동하는지 확인. (완료)

### B. 이미지 뷰어 (Image Viewer)
- [x] **파일 감지**: 이미지 확장자(.png, .jpg 등) 선택 시 마크다운 에디터 대신 이미지 전용 뷰어가 활성화되는지 확인. (완료)
- [x] **줌 컨트롤**: 줌 인/아웃 버튼 클릭 시 이미지 스케일이 정상적으로 변경되는지 확인. (완료)
- [x] **인터랙션**: 마우스 드래그를 통한 이미지 위치 이동(Panning) 기능 검증. (완료)

## 2. 테스트 환경
- **Target**: `Preview.web.tsx`, `ImageViewer.tsx`
- **Tool**: Playwright (Chromium, Firefox, WebKit)
- **Injection**: `__E2E_HOOKS__`를 통한 마크다운 컨텐츠 및 이미지 URI 주입.

## 3. 성공 기준 (Definition of Done)
- 마크다운 파싱 에러 없이 모든 지원 요소가 렌더링됨.
- 대용량 마크다운 문서에서도 미리보기 스크롤 네비게이션이 정확함.
- 이미지 뷰어의 확대/축소 및 이동 기능이 웹 환경에서 부드럽게 작동함.

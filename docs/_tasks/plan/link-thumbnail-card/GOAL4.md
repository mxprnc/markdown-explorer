# 링크 카드 버그 수정 및 로직 최적화 목표 (GOAL4.md)

구현된 링크 카드 기능에서 발견된 UI 버그를 수정하고, 상호작용 로직의 안정성을 높이는 것을 목표로 합니다.

---

## 🎯 핵심 목표

1.  **레이아웃 안정화 (Layout Stability)**:
    *   편집 팝업 내의 입력 필드(인풋)가 컨테이너 너비를 초과하거나 레이아웃이 깨지는 현상을 수정.
    *   긴 URL 입력 시에도 디자인 시스템이 유지되도록 `box-sizing` 및 여백 조정.

2.  **기능 신뢰성 확보 (Functional Reliability)**:
    *   웹 환경에서 '링크 복사' 버튼이 동작하지 않는 문제 해결 (`navigator.clipboard` 적용).
    *   '편집' 버튼 클릭 시 간헐적으로 무시되는 현상을 수정하여 클릭 응답성 개선.

3.  **링크 변환 UX 개선 (Auto-conversion UX)**:
    *   텍스트 형태의 URL이 `linkCard` 노드로 변환되는 시점을 최적화.
    *   커서가 링크 끝에 위치할 때도 자연스럽게 노드로 변환되어 프리미엄 UI 기능을 즉시 사용할 수 있도록 유도.

## ✅ 주요 체크리스트

- [x] **편집 팝업 인풋 요소 레이아웃 수정** (`box-sizing: border-box` 적용)
- [x] **링크 복사 기능 웹 호환성 확보** (Fallback 로직 추가)
- [x] **편집/타입 전환 버튼 이벤트 처리 강화** (클릭 이벤트 충돌 방지)
- [x] **LiveMarkdown URL 변환 로직 조정** (커서 위치에 따른 변환 감도 최적화)

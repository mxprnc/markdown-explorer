# 리렌더링 및 깜빡임 근본 해결 계획 (PLAN9.md)

기술적으로 더 깊은 수준에서의 렌더링 안정화 작업입니다.

---

## 🛠 단계별 구현 계획

### 1단계: VideoPlayer 렌더링 격리
- `VideoPlayer` 컴포넌트가 `LinkCardComponent`의 다른 상태(호버, 편집 모드 등) 변화에 반응하지 않도록 프롭스를 최소화하고 `React.memo`를 강화.
- `key` 값을 더 안정적인 고유값(URL)으로 고정하여 React가 이를 항상 동일한 인스턴스로 인식하게 함.

### 2단계: 메타데이터 관리 로직 고도화
- `getDerivedMetadata`가 불필요하게 객체를 새로 생성하지 않도록 메모이제이션 패턴 적용.
- `metadataCache`를 더 넓은 범위(또는 별도 유틸리티)에서 관리하여 모듈 재평가 시에도 유지되도록 검토.

### 3단계: 부모 레이아웃 영향 분석
- `EditorWorkspace`나 `app/index.tsx`에서 에디터 객체 자체가 새로 생성되는지 확인.
- 만약 에디터 인스턴스가 재생성된다면 모든 NodeView가 리셋되므로, 인스턴스 유지 로직 보강.

### 4단계: 시각적 보정 (Fallback)
- 비디오 리로드 시 검은 화면이 보이는 시간을 줄이기 위해 썸네일 이미지를 배경으로 깔아두는 기법 적용.
- 로딩 상태를 명시적으로 관리하여 부드러운 전환 유도.

# 실시간 변환 로직 안정화 계획 (PLAN8.md)

깜빡임 문제 해결을 위한 `appendTransaction` 로직 개선 및 최적화 단계입니다.

---

## 🛠 단계별 구현 계획

### 1단계: 기존 노드 존재 여부 확인 로직 추가
- `Editor.web.tsx`의 `LiveMarkdownExtension` 내부 루프 수정.
- 매칭된 URL 위치(`start`)에 대해 `newState.doc.nodeAt(start)`를 호출.
- 결과 노드가 `linkCard` 타입이고 URL이 동일하다면 변환 목록(`changes`)에서 제외.

### 2단계: 트랜잭션 병합 및 최소화
- 여러 개의 변환이 동시에 발생할 때 하나의 트랜잭션으로 확실히 묶여서 처리되는지 확인.
- `isTouching` 조건을 좀 더 보수적으로 적용하여 사용자 입력 방해 최소화.

### 3단계: 디버깅 및 예외 처리
- 변환이 일어날 때 콘솔 로그를 남겨 (개발 단계) 비정상적인 반복 호출이 있는지 확인.
- iCloud 등 외부 동기화로 인한 문서 업데이트 시의 동작 안정성 검토.

### 4단계: 검증
- 긴 문서나 복잡한 링크가 포함된 문서에서도 부드럽게 동작하는지 테스트.
- `GOAL8.md` 상태 업데이트.

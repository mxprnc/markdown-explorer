# 편집 팝업 최적화 목표 (GOAL6.md)

링크 카드의 타입(Text, Link, Card, Video)에 따라 편집 팝업의 입력 항목을 최적화하여 사용자 혼란을 방지하고 UI 일관성을 높이는 것을 목표로 합니다.

---

## 🎯 핵심 목표

1.  **타입별 맞춤형 UI (Context-aware UI)**:
    *   `Text` 모드: 에디터에 직접 노출되지 않는 '제목' 필드를 숨기고 URL 편집에 집중.
    *   `Link` / `Card` 모드: 닉네임(Alias)이나 제목이 중요하므로 '제목' 필드를 노출.
    *   `Video` 모드: 비디오 소스 URL에 집중할 수 있도록 UI 간소화.

2.  **사용자 인지 부하 감소 (Reduced Cognitive Load)**:
    *   불필요한 입력 항목을 제거하여 사용자가 현재 어떤 데이터를 수정하고 있는지 명확하게 인지하도록 함.
    *   모드 전환 시 편집 팝업의 구성이 자연스럽게 변경되도록 처리.

3.  **데이터 무결성 유지 (Data Integrity)**:
    *   UI에서 숨겨진 항목이라도 기존 데이터가 삭제되지 않도록 주의하여 관리.

## ✅ 주요 체크리스트

- [x] **'Text' 타입 선택 시 제목 입력란 숨기기**
- [x] **'Video' 타입 선택 시 제목 입력란 숨기기**
- [x] **'Link', 'Card' 타입 선택 시 모든 입력란 노출**
- [x] **타입 전환에 따른 팝업 UI 실시간 업데이트 확인**

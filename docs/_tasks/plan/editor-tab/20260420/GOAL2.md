# 탭 관리 고도화 목표 (GOAL2.md)

탭의 순서 변경 및 분할 창(Pane) 간 이동 기능을 구현하여 VSCode 수준의 자유로운 작업 환경을 제공합니다.

## 🎯 최종 목표
- [x] 동일한 Pane 내에서 탭을 드래그하여 자유롭게 순서를 변경할 수 있는가?
- [x] 탭을 드래그하여 Pane 1에서 Pane 2로(또는 그 반대로) 이동시킬 수 있는가?
- [x] 드래그 중 삽입 위치(Insertion Point)가 시각적으로 표시되는가?
- [x] 탭 이동 후 상태(OpenedFiles, PreviewFile 등)가 모든 Pane에서 정확히 동기화되는가?

## ✅ 성공 기준
1. **사용성**: 드래그 앤 드롭 동작이 끊김 없이 부드럽게 작동해야 함.
2. **데이터 무결성**: 탭 이동 시 파일 내용이나 관련 상태(스크롤 위치, 에디터 포커스 등)가 유실되지 않아야 함.
3. **시각적 피드백**: 드래그 중인 탭과 삽입될 위치가 사용자에게 명확히 인지되어야 함.
4. **예외 처리**: 자기 자신 위에 드롭하거나, 유효하지 않은 영역에 드롭했을 때 기존 상태가 안전하게 유지되어야 함.

## 🔴 미해결 이슈 및 제약 사항
- **빈 Pane 드롭 불가**: Split View 모드에서 한쪽 Pane이 비어 있을 때, 다른 Pane에서 탭을 드래그하여 이동시키는 기능이 완벽하게 동작하지 않음.
- **관련 문서**: [issue-20260420.md](../../../product/errors/split-view---editor-tab-drag-drop/issue-20260420.md)
- **해결 계획**: 추후 '미해결 이슈 수정 및 최적화' 기간에 브라우저 네이티브 이벤트 핸들링을 전면 재검토하여 수정 예정.

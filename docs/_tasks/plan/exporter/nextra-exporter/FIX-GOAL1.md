# GOAL: Nextra Export 기능 이슈 해결 (FIX-GOAL1)

## 🎯 핵심 목표
"Export to Nextra" 기능 수행 시, 선택한 디렉터리의 전체 구조가 반영되지 않고 기본 `index.md` 파일만 생성되는 문제를 해결합니다.

## 🔍 문제 현상
1. **디렉터리 트래버스 실패**: 선택된 노드의 하위 파일 및 폴더를 탐색하지 못함.
2. **데이터 구조 불일치**: UI 계층의 `FileSystemItem`(`kind` 속성 사용)과 Exporter 계층의 `ExportNode`(`type` 속성 사용) 간의 속성명 불일치로 인해 로직이 분기되지 않음.
3. **기본 파일만 생성**: 트래버스가 실패하면서 결과적으로 `ensureIndexExists` 로직에 의해 생성된 기본 `index.md`만 ZIP 파일에 포함됨.

## ✅ 성공 기준
- [ ] 선택한 디렉터리 내의 모든 마크다운 파일(`.md`, `.mdx`)이 Nextra 프로젝트 구조로 정상 추출되어야 함.
- [ ] 하위 디렉터리 구조가 유지되어야 하며, 각 디렉터리에 대한 `_meta.js`가 올바르게 생성되어야 함.
- [ ] 마크다운 내의 이미지 등 자산(Assets)이 `public/` 또는 지정된 경로로 정상 복사되어야 함.
- [ ] 마크다운이 아닌 일반 파일(예: `.ipynb`, `.txt`)도 선택적으로 포함되거나 무시되는 정책이 명확해야 함.

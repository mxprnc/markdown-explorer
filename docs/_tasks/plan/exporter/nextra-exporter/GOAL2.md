# Phase 2: Local Directory Scaffolding (Desktop) - Goals

이 문서는 데스크톱(PC) 환경에서 사용자의 로컬 파일 시스템에 직접 Nextra 프로젝트를 구축하는 단계의 목표를 정의합니다.

## 🎯 핵심 목표
1. **직접적인 파일 시스템 출력**: 브라우저의 ZIP 다운로드 방식에서 벗어나, 지정된 로컬 폴더에 실제 디렉토리와 파일을 직접 생성합니다.
2. **데스크톱 네이티브 경험**: OS 수준의 폴더 선택 대화상자(Folder Picker)를 연동하여 사용자 친화적인 경로 선택 기능을 제공합니다.
3. **자동화된 후속 작업 (Optional)**: 프로젝트 생성 후 자동으로 `npm install`을 수행하거나 개발 서버(`npm run dev`)를 실행할 수 있는 옵션을 제공합니다.
4. **대용량 처리 최적화**: 메모리 기반의 ZIP 압축 방식이 가진 한계를 넘어, 대규모 마크다운 위키 구조도 안정적으로 로컬에 구축합니다.

## ✅ 성공 기준 (Success Criteria)
- [ ] 데스크톱 앱 내에서 폴더 선택 버튼 클릭 시 OS 네이티브 폴더 선택창이 열린다.
- [ ] 선택된 경로에 `pages`, `public` 폴더 및 `next.config.mjs` 등 설정 파일이 정상 생성된다.
- [ ] 생성된 결과물이 로컬 터미널에서 `npm run dev` 실행 시 즉시 웹사이트로 구동된다.
- [ ] 웹 버전(ZIP)과 데스크톱 버전(Direct) 간의 엔진 코드가 공통 인터페이스를 통해 효율적으로 관리된다.

## 🚀 기대 효과
- 사용자가 내보낸 결과물을 즉시 편집하고 확인하는 워크플로우 단축.
- 로컬 개발 환경과의 직접적인 연동으로 실질적인 정적 사이트 생성기(SSG) 도구로서의 가치 확보.

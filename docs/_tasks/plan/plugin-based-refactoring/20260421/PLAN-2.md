# 플러그인 기반 템플릿 시스템 구현 계획 (PLAN-2.md)

로컬 마크다운 파일을 활용한 템플릿 플러그인 구현 단계별 계획입니다.

## 📅 실행 단계별 마일스톤

### Phase 1: TemplatesPlugin 클래스 및 폴더 구조 설계
- [x] `plugins/templates/TemplatesPlugin.ts`: 플러그인 클래스 생성.
- [x] 앱 로드 시 `.mark-explorer/templates/` 폴더가 없으면 자동으로 생성하는 로직 구현.

### Phase 2: 템플릿 탐색 및 로딩 엔진 개발
- [x] `Vault` API를 활용하여 지정된 폴더 내의 모든 `.md` 파일 목록 스캔.
- [x] 각 파일의 파일명을 템플릿 이름으로, 본문을 템플릿 내용으로 매핑하는 데이터 구조 설계.

### Phase 3: 변수 치환 및 삽입 로직 구현
- [x] **변수 파서**: `{{date}}`, `{{time}}`, `{{title}}` 등 미리 정의된 키워드를 실시간 데이터로 치환하는 유틸리티 작성.
- [x] **에디터 주입**: 현재 활성화된 에디터의 커서 위치에 치환된 마크다운 텍스트를 삽입하는 API 연동.

### Phase 4: UI 및 명령어 연동
- [x] **Insert Template 명령어**: 명령어 레지스트리에 등록하여 단축키(`Alt + T`)로 호출 가능하게 함.
- [x] **템플릿 선택기(QuickPicker)**: 사용자가 등록된 템플릿 중 하나를 선택할 수 있는 모달 UI 연결.

### Phase 5: 최종 검증 및 테스트
- [x] **단위 테스트 작성**: `MarkdownUtils.test.ts`를 통해 변환 로직 및 변수 치환 검증 완료.
- [x] **에디터 정규화**: 불러오기/저장 시 헤딩 및 링크 이스케이프 문제 해결을 위한 전후 처리 적용.
- [x] **안정성 확보**: 에디터 포커스 상태 시 업데이트 충돌 방지 로직 적용.

## ⚠️ 기술적 고려사항
- **파일 경로 처리**: 상대 경로(`./`)와 절대 경로 간의 혼선을 방지하기 위해 `Vault` 인터페이스 활용 필수.
- **치환자 충돌 방지**: 사용자가 작성한 본문 내용 중 우연히 `{{ }}` 형식이 포함될 경우를 대비한 예외 처리 고민.

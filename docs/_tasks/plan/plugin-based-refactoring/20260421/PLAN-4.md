# AI 기반 보안 자동화 구현 계획 (PLAN-4.md)

Gemini CLI/API를 활용한 프로젝트 특화 보안 감사 시스템 구축 계획입니다.

## 📅 실행 단계별 마일스톤

### Phase 1: 보안 감사 프롬프트 엔지니어링
- [ ] 보안 가이드라인(`coding-style.md`)과 마크다운 환경의 특이사항을 반영한 시스템 프롬프트 설계.
- [ ] 위험 등급 분류 및 리포트 출력 형식(Markdown) 정의.

### Phase 2: 보안 감사 스크립트 개발
- [ ] `scripts/security-audit.ts`: Gemini API를 호출하여 특정 파일의 보안을 리뷰하는 스크립트 작성.
- [ ] `git diff`를 연동하여 마지막 커밋 이후 변경된 파일만 추출하는 로직 구현.

### Phase 3: 로컬 개발 환경 연동
- [ ] `npm run audit:ai` 명령어를 통해 개발자가 수동으로 보안 검사를 수행할 수 있도록 설정.
- [ ] 발견된 취약점을 터미널에 요약 출력하는 기능.

### Phase 4: GitHub Actions 및 CI/CD 통합
- [ ] GitHub 워크플로우 파일 생성: PR 생성 시 자동으로 스크립트를 실행하고 리뷰 코멘트를 남기는 기능.
- [ ] 보안 등급이 'High'인 경우 병합(Merge)을 차단하는 체크 로직 검토.

### Phase 5: 모니터링 및 고도화
- [ ] AI의 오탐(False Positive) 사례를 수집하여 프롬프트 지속 업데이트.
- [ ] MCP 서버 연동 시 발생할 수 있는 네트워크 보안 검사 규칙 추가.

## ⚠️ 기술적 고려사항
- **API 키 관리**: CI 환경에서 Gemini API 키가 노출되지 않도록 GitHub Secrets 활용.
- **토큰 제한**: 코드 양이 많을 경우 토큰 제한을 고려하여 파일별 또는 청크(Chunk)별로 나누어 분석.
- **결과 신뢰성**: AI의 결과는 항상 개발자의 최종 검토를 거쳐야 함을 가이드라인에 명시.

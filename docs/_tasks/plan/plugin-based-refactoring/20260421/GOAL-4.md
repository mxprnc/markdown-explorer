# AI 기반 보안 자동화 및 감사 시스템 구축 목표 (GOAL-4.md)

전통적인 보안 도구의 한계를 넘어, Gemini AI의 코드 이해 능력을 활용하여 프로젝트의 논리적 결함과 보안 취약점을 자동으로 탐지하는 시스템을 구축합니다.

## 🎯 최종 목표
- [ ] **AI 보안 스크립트**: Gemini API를 사용하여 소스 코드를 보안 관점에서 분석하는 실행 가능한 스크립트가 존재하는가?
- [ ] **문맥 기반 취약점 탐지**: 플러그인 권한 오용, 자원 해제 누락 등 로직상의 보안 결함을 찾아낼 수 있는가?
- [ ] **CI/CD 통합**: GitHub Actions를 통해 PR(Pull Request) 시 자동으로 보안 리뷰 리뷰 결과가 생성되는가?
- [ ] **보안 리포트 생성**: 발견된 이슈의 위험도(High/Medium/Low)와 수정 권장 사항이 포함된 리포트를 제공하는가?

## ✅ 성공 기준
1. **정밀도**: 단순히 키워드를 찾는 수준을 넘어, 코드의 실행 흐름 상에서 발생할 수 있는 보안 위험을 지적해야 함.
2. **자동화**: 개발자가 일일이 명령어를 실행하지 않아도 Git Push/PR 시점에 자동으로 검사가 이루어져야 함.
3. **학습성**: `coding-style.md`와 같은 프로젝트 내부 문서를 컨텍스트로 활용하여 프로젝트 특화된 보안 검사를 수행해야 함.
4. **효율성**: 전체 코드가 아닌 변경된 파일(Diff)만 집중적으로 분석하여 API 비용과 처리 시간을 최적화해야 함.

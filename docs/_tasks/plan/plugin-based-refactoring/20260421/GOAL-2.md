# 플러그인 기반 템플릿 시스템 구축 목표 (GOAL-2.md)

사용자가 반복적으로 사용하는 마크다운 형식을 파일로 저장하고, 필요할 때마다 현재 문서에 즉시 삽입할 수 있는 확장 가능한 템플릿 시스템을 구축합니다.

## 🎯 최종 목표
- [x] **로컬 파일 기반 저장**: `.mark-explorer/templates/` 폴더 내의 `.md` 파일을 템플릿으로 인식하는가?
- [x] **플러그인화**: 기존 코어 엔진을 수정하지 않고 `TemplatesPlugin`이라는 독립 모듈로 구현되었는가?
- [x] **명령어 연동**: 명령어 팔레트나 전용 버튼을 통해 템플릿 목록을 호출하고 삽입할 수 있는가?
- [x] **동적 변수 치환**: `{{date}}`, `{{time}}` 등 동적 텍스트 치환 기능이 작동하는가?

## ✅ 성공 기준
1. **데이터 영속성**: 브라우저 종료 후에도 사용자가 작성한 템플릿 파일이 안전하게 보존되어야 함.
2. **사용자 편의성**: 복잡한 설정 없이 특정 폴더에 마크다운 파일을 넣는 것만으로 템플릿이 자동으로 등록되어야 함.
3. **유연성**: 플러그인을 비활성화하면 템플릿 관련 기능과 UI가 완벽히 제거되어야 함.
4. **확장성**: 향후 유튜브 API 연동 등을 통해 특정 필드를 자동으로 채워주는 기능을 추가할 수 있는 구조여야 함.

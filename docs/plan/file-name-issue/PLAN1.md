# 비영어권 파일명 경로 처리 구현 계획 (PLAN1.md)

기술적인 파일명 정제 및 경로 관리 로직을 구현하기 위한 세부 계획입니다.

---

## 🛠 처리 로직 상세

### 1. 한글 자모 분리 방지 (NFC 정규화)
- **문제**: Mac 환경에서 업로드된 파일명은 'ㅎㅏㄴ'처럼 분리되어 저장될 수 있음.
- **해결**: JavaScript의 `.normalize('NFC')`를 사용하여 모든 비영어권 문자열을 표준 형식으로 통합.

### 2. 파일명 Sanitization (정제)
- **규칙**:
    - 공백 및 연속된 특수문자는 하이픈(`-`)으로 치환.
    - 대문자는 소문자로 변환 (Case-insensitive 호환성).
    - 허용되는 문자: `a-z`, `0-9`, `가-힣`, `.`, `-`, `_`.
- **예시**: `내 이미지 @2026 테스트.png` -> `내-이미지-2026-테스트.png`

### 3. 모바일 경로 구조 설계
- **구조**: `./img/{상위-파일명-슬러그}/{순번}.png`
- **구현**: 원본 파일명이 너무 복잡할 경우 슬러그화하여 폴더명으로 사용하고, 내부 이미지는 인덱싱하여 관리함으로써 경로 복잡도를 낮춤.

## 💻 권장 구현 코드 (Snippet)
```javascript
function sanitizePathName(input) {
  return input
    .toLowerCase()
    .normalize('NFC')
    .replace(/\s+/g, '-') // 공백 -> 하이픈
    .replace(/[^a-z0-9가-힣ㄱ-ㅎㅏ-ㅣ.\-_]/g, '-') // 특수문자 제거
    .replace(/-+/g, '-') // 연속 하이픈 통합
    .replace(/^-+|-+$/g, ''); // 앞뒤 하이픈 제거
}
```

## ⚠️ 주의 사항
- **URL Encoding**: 마크다운 링크 내에 한글이 포함될 경우, 일부 환경에서는 `%EC%9D%B4...` 처럼 인코딩된 경로가 필요할 수 있으므로 렌더링 시 이를 고려해야 함.
- **중복 처리**: 동일한 슬러그 이름이 발생할 경우 타임스탬프 등을 추가하여 덮어쓰기 방지.

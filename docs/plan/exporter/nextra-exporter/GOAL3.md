# Phase 3: Automated Dev Server & Browser Open (App Only) - Goals

이 문서는 데스크톱 앱 환경에서 Nextra 프로젝트 생성 직후, 개발 서버를 자동으로 실행하고 브라우저를 오픈하여 사용자에게 즉각적인 결과물을 보여주는 자동화 단계의 목표를 정의합니다.

## 🎯 핵심 목표
1. **Zero-Config 실행**: 사용자가 터미널을 열고 명령어를 입력할 필요 없이, 버튼 클릭 한 번으로 `npm install` 및 `npm run dev`가 완료되도록 합니다.
2. **실시간 상태 피드백**: 개발 서버가 준비되는 과정(의존성 설치, 컴파일 등)을 앱 내부 UI에서 실시간으로 확인할 수 있게 합니다.
3. **지능형 브라우저 오픈**: 서버가 준비되는 즉시 시스템 기본 브라우저를 호출하여 `http://localhost:3000`을 자동으로 엽니다.
4. **프로세스 관리**: 앱이 종료되거나 내보내기 마법사가 닫힐 때, 백그라운드에서 실행 중인 개발 서버 프로세스를 안전하게 종료(Cleanup)합니다.

## ✅ 성공 기준 (Success Criteria)
- [ ] 내보내기 완료 후 "개발 서버 실행" 체크박스가 선택되어 있으면 자동으로 터미널 프로세스가 시작된다.
- [ ] 앱 내부의 로그 뷰어(Log Viewer)를 통해 `next dev`의 출력이 실시간으로 표시된다.
- [ ] 포트 3000번이 활성화되면 브라우저가 자동으로 실행되어 생성된 사이트를 보여준다.
- [ ] Node.js나 npm이 설치되어 있지 않은 환경의 경우, 적절한 에러 메시지와 함께 수동 가이드를 제시한다.

## 🚀 기대 효과
- 기술적 지식이 부족한 사용자도 자신의 문서를 즉시 웹사이트로 확인 가능.
- 내보내기 → 터미널 이동 → 명령어 입력 → 대기 → 브라우저 오픈으로 이어지는 5단계 과정을 1단계로 축소.

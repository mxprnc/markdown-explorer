# 템플릿 전용 UI 시스템 구축 목표 (GOAL-3.md)

사용자가 템플릿을 더 직관적으로 관리하고 사용할 수 있도록, 사이드바 전용 뷰와 관리 도구를 포함한 시각적 인터페이스를 구축합니다.

## 🎯 최종 목표
- [ ] **사이드바 전용 탭**: 파일 탐색기와 별개로 템플릿만 모아볼 수 있는 전용 탭이 존재하는가?
- [ ] **템플릿 리스트 뷰**: 등록된 모든 템플릿을 아이콘과 함께 직관적인 목록으로 표시하는가?
- [ ] **원클릭 액션**: 목록에서 즉시 삽입(Insert)하거나 편집(Edit)할 수 있는 버튼이 제공되는가?
- [ ] **신규 생성 워크플로우**: UI 내의 '+' 버튼을 통해 즉시 새로운 템플릿 파일을 생성할 수 있는가?

## ✅ 성공 기준
1. **접근성**: 명령어 팔레트뿐만 아니라 사이드바 탭을 통해 마우스만으로도 모든 템플릿 기능을 사용할 수 있어야 함.
2. **실시간 동기화**: 로컬 폴더(`.mark-explorer/templates`)에 파일을 추가하거나 삭제하면 UI 목록에 즉시 반영되어야 함.
3. **사용자 경험(UX)**: 템플릿 편집 시 현재 편집 중인 문서를 방해하지 않고 자연스럽게 탭이 전환되어야 함.
4. **일관된 디자인**: 기존 탐색기(File Explorer)와 통일된 디자인 시스템을 유지하여 이질감을 최소화함.

# 실시간 변환 로직 안정화 목표 (GOAL8.md)

링크 카드로의 실시간 변환 과정에서 발생하는 깜빡임(Flickering) 문제를 해결하고, 에디터의 성능과 안정성을 확보하는 것을 목표로 합니다.

---

## 🎯 핵심 목표

1.  **중복 변환 방지 (Idempotent Conversion)**:
    *   이미 `linkCard` 노드로 변환된 텍스트에 대해 중복으로 변환 트랜잭션이 발생하지 않도록 로직을 수정하여 무한 루프 가능성 차단.

2.  **랜더링 사이클 최적화 (Rendering Cycle Optimization)**:
    *   불필요한 트랜잭션 발생을 억제하여 NodeView의 불필요한 재마운트(Remount)를 방지하고 시각적 깜빡임 제거.

3.  **정교한 위치 감지 (Precise Position Detection)**:
    *   현재 커서 위치와 변환 대상 영역 간의 관계를 더 정확히 계산하여, 편집 중인 텍스트가 의도치 않게 노드로 변환되거나 튀는 현상 방지.

## ✅ 주요 체크리스트

- [ ] **변환 전 해당 위치의 노드 타입 확인 로직 추가 (nodeAt)**
- [ ] **동일한 URL에 대한 반복 변환 시도 차단**
- [ ] **트랜잭션 발생 조건 최소화 (실제 변경이 필요한 경우만 수행)**
- [ ] **에디터 성능 모니터링 및 깜빡임 유무 재검증**

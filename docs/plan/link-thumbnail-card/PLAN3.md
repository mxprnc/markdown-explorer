# 링크 편집 및 툴바 구현 계획 (PLAN3.md) - ✅ 완료

사용자가 '편집' 기능을 더 쉽게 찾고 사용할 수 있도록 호버 동작과 툴바 구성을 리팩토링합니다.

---

## 🛠 단계별 구현 계획

### 1단계: 호버 동작 및 툴바 통합
- **로직 변경**: 기존에 클릭(`showMenu`)으로만 노출되던 '링크 툴바'를 `isHovered` 상태일 때 노출되도록 변경.
- **UI 통합**:
    - 상단: 타입 선택기 (Text, Link, Card, Video)
    - 하단: 링크 툴바 (URL, 복사, 편집)
    - 두 요소를 하나의 논리적 그룹으로 묶어 호버 시 함께 관리.

### 2단계: Notion 스타일 디자인 고도화
- **툴바 스타일**:
    - 배경: Glassmorphism (`backdrop-filter`) 유지.
    - URL 표시: `JetBrains Mono` 폰트, `max-width` 설정을 통한 자동 생략.
    - 버튼: 아이콘과 '편집' 텍스트를 함께 배치하여 가독성 증대.
- **편집 팝업**:
    - 툴바에서 '편집' 버튼 클릭 시 나타나는 팝업의 `z-index`와 위치를 최적화하여 툴바와 겹치지 않게 조정.

### 3단계: 상호작용 및 예외 처리
- **Hover Bridge**: 링크 노드의 상하단에 투명한 패딩 영역을 확보하여 마우스 이동 시 안정성 확보.
- **편집 상태 유지**: 편집 팝업(`isEditing`)이 열려 있을 때는 호버 여부와 상관없이 UI가 유지되도록 처리.
- **에디터 포커스**: 편집 팝업의 입력 필드에 포커스가 갈 때 에디터 본문의 커서가 방해되지 않도록 `preventDefault` 처리.

### 4단계: 테스트 및 완료 업데이트
- 실제 호버 시의 반응 속도 및 시각적 자연스러움 확인.
- `GOAL3.md` 및 `PLAN3.md`에 진행 완료 상황 업데이트.

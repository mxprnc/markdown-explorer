# UI Component E2E Test Goals

이 문서는 `components/ui` 내의 공통 UI 컴포넌트들에 대한 E2E 테스트 최종 목표를 정의합니다.

## 1. 핵심 검증 목표

### A. QuickPicker (명령/템플릿 피커)
- [x] **가시성 및 포커스**: 모달 오픈 시 검색 필드에 자동으로 포커스가 가는지 확인. (완료)
- [x] **필터링 로직**: 검색어 입력 시 아이템 리스트가 실시간으로 필터링되는지 확인. (완료)
- [x] **선택 및 동작**: 아이템 클릭 또는 엔터 키 입력 시 `onSelect` 핸들러가 실행되고 모달이 닫히는지 확인. (완료)
- [x] **예외 상태**: 결과가 없을 때 "No results found" 안내 문구가 정상 노출되는지 확인. (완료)

### B. Collapsible (아코디언 컴포넌트)
- [x] **상태 전환**: 헤딩 클릭 시 컨텐츠 영역이 펼쳐지거나 닫히는지 확인. (완료)
- [x] **시각적 피드백**: 상태에 따라 아이콘(Chevron)이 올바르게 회전하는지 확인. (완료)

### C. Button (공통 버튼)
- [x] **상태 대응**: `disabled` 속성 적용 시 클릭이 방지되고 시각적으로 불투명도가 조절되는지 확인. (완료)
- [x] **변형(Variant) 확인**: Primary, Secondary 등 스타일 변형이 테마에 맞게 올바르게 적용되는지 확인. (완료)

## 2. 테스트 환경
- **Target**: `QuickPicker.tsx`, `Collapsible.tsx`, `Button.tsx`
- **Tool**: Playwright (Chromium)
- **Context**: 실제 서비스 내 사용 사례(템플릿 피커 호출 등)를 활용한 검증.

## 3. 성공 기준 (Definition of Done)
- 모든 공통 UI 컴포넌트가 디자인 시스템 가이드라인에 따라 일관되게 동작함.
- 키보드 인터랙션(엔터, 닫기 등)이 웹 표준에 맞게 작동함.
- 테마(다크/라이트) 전환 시 UI 요소의 시인성이 확보됨.

# Core Components E2E Test Goals

이 문서는 에디터(`Editor`), AI 채팅(`GeminiChat`), 레이아웃(`TabBar` 등) 핵심 컴포넌트들에 대한 E2E 테스트 최종 목표를 정의합니다.

## 1. 핵심 검증 목표

### A. Editor (CodeMirror 기반 에디터)
- [x] **텍스트 입력 및 유지**: 에디터에 타이핑한 내용이 상태에 반영되고, 탭 전환 후에도 유지되는지 확인.
- [x] **에디터-미리보기 동기화**: 에디터 내용 수정 시 실시간으로 미리보기(Preview) 화면이 갱신되는지 확인.
- [x] **자동 저장/변경 감지**: 내용 수정 후 다른 파일 선택 시 기존 내용이 메모리에 정상적으로 보존되는지 확인.

### B. GeminiChat (AI 어시스턴트)
- [x] **채팅 인터랙션**: 메시지 입력 후 전송 시 메시지 버블이 생성되고 로딩 상태가 표시되는지 확인.
- [x] **모드 전환**: General 모드와 Archive 모드 전환 시 UI(아카이브 경로 입력창 등)가 올바르게 변경되는지 확인.
- [x] **설정 연동**: 설정 버튼 클릭 시 Gemini 설정 모달이 호출되는지 확인.

### C. TabBar & Layout (내비게이션)
- [x] **탭 관리**: 여러 파일을 열었을 때 탭이 생성되고, 클릭 시 해당 파일로 에디터가 전환되는지 확인.
- [x] **탭 닫기/핀 고정**: 탭 닫기 버튼 작동 여부 및 프리뷰 탭의 영구 탭 전환(Pin) 로직 확인.
- [x] **사이드바 전환**: 탐색기 외의 다른 사이드바 뷰(등록된 플러그인 등)로의 전환이 정상적인지 확인.

## 2. 테스트 환경
- **Target**: `Editor.web.tsx`, `GeminiChat.tsx`, `TabBar.tsx`, `Sidebar.tsx`
- **Tool**: Playwright (Chromium)
- **Data**: 가상 파일 시스템(Mock FS)을 활용한 다중 파일 편집 시나리오.

## 3. 성공 기준 (Definition of Done)
- 에디터의 편집 내용이 손실 없이 미리보기 및 시스템 상태와 완벽히 동기화됨.
- AI 채팅 인터페이스가 모든 모드에서 사용자 입력을 정상적으로 처리함.
- 탭 시스템이 VSCode 수준의 부드러운 전환과 상태 유지를 보장함.

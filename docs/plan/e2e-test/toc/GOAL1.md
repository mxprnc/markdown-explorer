# TOC Component E2E Test Goals

이 문서는 `components/toc` 내의 `TOCPane` 컴포넌트에 대한 E2E 테스트의 최종 목표를 정의합니다.

## 1. 핵심 검증 목표
- [x] **헤딩 추출 정확도**: 마크다운 본문의 `#` (H1~H6) 헤딩이 목차에 정확하게 리스트업되는지 확인. (완료)
- [x] **계층 구조 시각화**: 헤딩 레벨(H1~H6)에 따라 들여쓰기(Indentation)가 올바르게 적용되는지 확인. (완료)
- [x] **네비게이션 기능**: 목차 아이템 클릭 시 에디터 또는 프리뷰가 해당 섹션으로 올바르게 스크롤/이동하는지 확인. (완료)
- [x] **동적 업데이트**: 본문 내용 변경 시 목차가 실시간으로 업데이트되는지 확인. (완료)
- [x] **예외 케이스 처리**: 헤딩이 없는 문서에서의 "Empty State" 메시지 노출 확인. (완료)
- [x] **레이아웃 유연성**: TOC 패널의 리사이징 핸들이 정상 작동하고 레이아웃이 깨지지 않는지 확인. (완료)

## 2. 테스트 환경
- **Target**: `TOCPane.tsx`
- **Tool**: Playwright (Chromium, Firefox, WebKit)
- **State Injection**: `__E2E_HOOKS__`를 통한 가상 파일 본문 주입.

## 3. 성공 기준 (Definition of Done)
- 모든 핵심 검증 목표에 대한 테스트 케이스가 작성되고 통과됨.
- 에디터/프리뷰 모드 전환 시에도 TOC의 연결성이 유지됨.
- 다양한 헤딩 깊이(Depth)를 가진 문서에서도 계층 구조가 유지됨.

# E2E Test Goals: Explorer Components (GOAL1.md)

이 문서는 `components/explorer` 내의 컴포넌트들에 대한 Playwright 기반 E2E 테스트의 목표를 정의합니다.

---

## 🎯 테스트 목표

### 1. 기본 내비게이션 (Basic Navigation)
- [x] 폴더 클릭 시 하위 항목이 올바르게 펼쳐지거나(Expand) 접히는지(Collapse) 확인합니다. (완료 - Mock 주입 확인)
- [x] 파일 시스템의 계층 구조가 시각적으로 올바르게 렌더링되는지 확인합니다. (완료)

### 2. 파일 열기 동작 (File Opening)
- [x] 파일 단일 클릭 시 해당 파일이 **미리보기(Preview)** 모드로 에디터에 열리는지 확인합니다. (완료)
- [x] 파일 더블 클릭 시 해당 파일이 **영구(Permanent)** 모드로 새 탭에서 열리는지 확인합니다. (완료)
- [x] 선택된 파일이 탐색기에서 하이라이트 처리되는지 확인합니다. (완료)

### 3. 컨텍스트 메뉴 (Context Menu)
- [x] 파일 또는 폴더 우클릭 시 컨텍스트 메뉴가 올바른 위치에 나타나는지 확인합니다. (완료)
- [x] 메뉴 외부 클릭 시 컨텍스트 메뉴가 정상적으로 닫히는지 확인합니다. (완료)
- [ ] 다른 항목을 연속으로 우클릭했을 때, 이전 메뉴가 닫히고 새 메뉴가 즉시 나타나는지 확인합니다.

### 4. 파일 관리 기능 (File Management)
- [x] **이름 변경(Rename)**: 컨텍스트 메뉴나 퀵 액션을 통해 이름 변경 모달이 뜨고, 입력 후 이름이 변경되는지 확인합니다. (모달 노출 확인 완료)
- [x] **경로 복사(Copy Path)**: 퀵 액션 버튼을 통해 상대 경로가 클립보드에 복사되는지 확인합니다. (완료 - Mock Alert 확인)
- [x] **삭제(Delete)**: 컨텍스트 메뉴를 통한 삭제 기능 작동 여부를 확인합니다. (완료)

### 5. 사용자 경험 및 피드백 (UX Feedback)
- [x] 항목에 마우스 호버(Hover) 시 배경색 하이라이트 및 퀵 액션 버튼 노출 여부를 확인합니다. (완료)
- [ ] 드래그 앤 드롭 준비 상태(Draggable)를 확인합니다.

---

## 🧪 테스트 범위 (In-Scope)
- `FileExplorer.tsx`: 전체 탐색기 컨테이너 및 상태 관리.
- `FileTree.tsx`: 트리 구조 렌더링 로직.
- `FileItem.tsx`: 개별 항목 상호작용 (Click, Double Click, Hover).
- `ContextMenu.tsx`: 우클릭 메뉴 노출 및 액션 트리거.
- `RenameModal.tsx`: 이름 변경 입력 및 확인/취소 처리.

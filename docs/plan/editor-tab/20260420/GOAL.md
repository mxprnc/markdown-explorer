# Goal: VSCode-Style Tab System Implementation

## 🎯 목적
VSCode의 지능적인 탭 관리 시스템을 벤치마킹하여, 탐색기(Explorer)와 에디터 간의 상호작용 속도를 높이고 불필요하게 탭이 쌓이는 현상을 방지하며 강력한 파일 제어 기능을 제공합니다.

## 🚀 주요 상호작용 목표

### 1. 프리뷰 탭 (Single Click)
- **사용자 경험**: "잠시 훑어보기" 모드.
- **동작**: 탐색기에서 파일을 한 번 클릭하면 해당 파일이 **이탤릭체 제목**의 탭으로 열립니다. 이미 다른 파일이 프리뷰 모드로 열려 있다면, 해당 탭의 내용만 교체됩니다.
- **가치**: 수많은 파일을 빠르게 클릭하며 내용을 확인할 때 탭이 무분별하게 생성되는 것을 방지합니다.

### 2. 영구 탭 (Double Click / Edit)
- **사용자 경험**: "본격적으로 작업하기" 모드.
- **동작**:
    - 파일을 더블 클릭하면 해당 파일이 **일반체 제목**의 탭으로 고정되며 열립니다.
    - 프리뷰 상태의 탭에서 내용을 수정하기 시작하면 자동으로 영구 탭으로 전환됩니다.
- **가치**: 사용자가 의도한 핵심 작업 파일들을 명확히 구분하여 유지합니다.

### 3. 기능적인 탭 컨텍스트 메뉴 (Right Click)
- **기능 리스트**:
    - **Close**: 현재 탭 닫기
    - **Close Others**: 현재 탭을 제외한 모든 탭 닫기
    - **Close Saved**: 저장된 파일들만 골라서 닫기 (Dirty flag 없는 파일)
    - **Close All**: 모든 탭 닫기
    - **Copy Relative Path**: 프로젝트 루트 기준 상대 경로 복사
    - **Pin**: 프리뷰 탭을 영구 탭으로 승격/고정
    - **Reveal in Finder**: 실제 OS의 파일 탐색기에서 해당 파일 위치 열기

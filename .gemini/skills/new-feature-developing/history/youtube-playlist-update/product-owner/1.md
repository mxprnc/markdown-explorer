# [Phase 1] Planning & Idea Storming
**Feature**: youtube-playlist-update
**Version**: 1
**Date**: 2026-05-04

## Background (요구사항 요약)
- **목표**: Extract Playlist 시 대상 마크다운 파일에 사용자가 작성한 기존 텍스트가 혼합되어 있는 경우, 기존 텍스트는 보존하고 플레이리스트 부분만 최신 상태로 업데이트하는 기능.
- **관련 문서**: `docs/plan/exporter/youtube-playlist-exporter/prompt1.md` (Idea 4)

## Idea Proposals

### Idea A: 마크다운 주석 기반 영역 지정 (Block Replacement)
- **개념**: 플레이리스트가 삽입되는 영역을 마크다운 주석(예: `<!-- playlist-start: p_A -->` ... `<!-- playlist-end -->`)으로 감싸서 구분하고, 업데이트 시 해당 영역 내부만 교체합니다.
- **장점**: 파싱 로직이 매우 단순하고 확실하며, 업데이트할 영역을 명확히 식별할 수 있습니다.
- **단점**: 사용자가 텍스트를 편집하다가 실수로 주석을 지우거나 훼손하면 업데이트 로직이 고장날 수 있습니다.

### Idea B: 스마트 URL 패턴 매칭 (Smart Pattern Matching)
- **개념**: 별도의 주석이나 태그 없이, 파일 내에 연속적으로 존재하는 YouTube URL 블록(또는 특정 패턴의 리스트)을 식별하여 해당 부분만 새로운 플레이리스트 목록으로 치환합니다.
- **장점**: 마크다운 원본이 가장 깔끔하게 유지되며, 사용자가 특수 문법이나 주석을 전혀 신경 쓸 필요가 없습니다.
- **단점**: 플레이리스트와 무관한 일반 유튜브 링크가 파일 내에 섞여있을 경우 파싱 오류나 원치 않는 링크 덮어쓰기(엣지 케이스)가 발생할 위험이 있습니다.

### Idea C: 시각적 머지 툴 (Interactive Merge & Preview)
- **개념**: 백그라운드에서 바로 덮어쓰기를 하지 않고, 마치 Git에서 충돌을 해결하듯 '기존 문서'와 '업데이트된 목록'을 모달이나 Split View 형태의 UI로 띄워 사용자가 최종 확인 후 반영(Confirm)하게 합니다.
- **장점**: 사용자가 직접 작성한 소중한 텍스트가 실수로 유실될 위험을 원천 차단하며, '프리미엄한 사용자 경험(UX)'을 제공할 수 있습니다.
- **단점**: 매 업데이트 시 사용자의 추가적인 확인(클릭) 작업이 요구되어 번거로울 수 있으며, UI/컴포넌트 개발 공수가 큽니다.

### Idea D: 구조화된 마크다운 템플릿 (Structured Markdown Binding) [✨ 사용자 제안 바탕]
- **개념**: 단순 URL 나열이 아니라, 처음 Export할 때부터 영상별로 `제목-링크-메모영역`이 하나의 "블록"으로 묶이도록 마크다운 구조를 설계합니다. 모달에서 순서를 바꾸거나 삭제할 때 이 "블록 단위"로 데이터를 조작합니다.
- **파생 안건 (D-1, D-2, D-3)**:
  - **D-1 (헤딩 기반)**: `## 제목` / `링크` / `### Note` / `---` 구조. 직관적이나 TOC(목차)가 너무 길어질 우려가 있습니다.
  - **D-2 (리스트+인용구 기반)**: `- **제목**` / `링크` / `> 메모영역` 구조. TOC를 오염시키지 않고 들여쓰기로 묶여 있어 파싱이 용이합니다.
  - **D-3 (주석 바인딩)**: `<!-- item: id -->` / `제목+링크` / `자유로운 메모` / `<!-- item-end -->` 구조. 사용자의 자유도가 가장 높습니다.
- **장점**: 영상 정보와 사용자의 메모가 논리적/시각적으로 완벽히 결합(Binding)되어, 순서 변경 시 메모가 누락될 걱정이 없습니다. 파서(Parser) 구현이 매우 명확해집니다.
- **단점**: 기존에 단순히 URL만 나열해둔 플레이리스트와는 호환성을 맞춰야 하는 추가 작업이 필요할 수 있습니다.

### Idea E: Export 이원화 전략 (Dual Export Strategy) [✨ 사용자 제안 바탕]
- **개념**: 사용자의 목적에 따라 템플릿 구조를 선택적으로 적용하고, 각각에 맞는 전용 업데이트 로직을 분리하는 전략입니다.
- **구조**:
  1. **Youtube Playlist Markdown (전용 문서 모드)**: 
     - **적용**: D-1(헤딩 기반) 채택. 
     - **목적**: 오직 해당 플레이리스트만을 관리하고 시청 노트를 정리하기 위해 생성된 '전용 문서'로 추출.
     - **특징**: 각 영상이 하나의 큰 챕터(`##`)가 되며, UI에서도 전용 파서/업데이터가 동작함.
  2. **Playlist Link List (혼합 모드)**: 
     - **적용**: D-2(리스트+인용구 기반) 채택.
     - **목적**: 사용자가 이미 작성 중이던 일반 문서 중간에 플레이리스트 정보를 레퍼런스 용도로 '삽입/추가'할 때 사용.
     - **특징**: 기존 마크다운 구조(TOC 등)를 훼손하지 않으며, 업데이트 시 해당 리스트 블록 내에서만 조용히 갱신됨.

### [Idea H] 정직한 미리보기(Honest Preview) 및 정밀한 추출 제어
사용자에게 정직한 피드백을 제공하고, 추출 환경을 더욱 정밀하게 제어할 수 있도록 개선합니다.

1.  **2-Tab 시스템 (Markdown | Preview)**:
    *   **Markdown**: 실제 삽입될 원시 마크다운 코드를 보여줍니다.
    *   **Preview (Honest)**: `MarkdownPreview`를 사용하여 **실제 파일에 저장될 내용을 100% 동일하게 시각화**합니다.
2.  **Item Limit 수동 입력 기능**:
    *   기존의 고정된 선택 방식에서 벗어나, 사용자가 원하는 개수를 직접 입력할 수 있는 **TextInput** 방식으로 복구했습니다.
    *   **기본값**: `20`개로 설정하여 범용성을 확보했습니다.
3.  **List Type 기본값 최적화**:
    *   **기본값**: `Plain`으로 설정하여 기호 없는 깔끔한 텍스트 나열을 기본으로 제공합니다.
4.  **다이얼로그 사이즈 확대**:
    *   더 넓은 작업 공간과 프리뷰 확인을 위해 모달의 크기를 화면의 **90% x 85%** 수준으로 대폭 확대했습니다. (최대 너비 1200px)

  - **결정 사항**:
    - **파일명**: `{재생목록 제목}_{번호}.md` 포맷 사용.
    - **내부 구조**: 각 영상 아이템은 `## {재생목록 아이템 제목}` 헤딩으로 시작하며, 하위에 선택된 포맷(Text, URL, Card, Video)에 맞게 링크 내용이 삽입되어야 함.
    - **포맷 선택기 복구**: 이전에 기획되었던 `Text | URL | Card | Video` 포맷 선택 메뉴와 더불어, 리스트 형태를 결정하는 `List Type (Numbered | Bulleted | Plain)` 옵션이 Expanded 모달에도 반드시 포함되어야 함.
    - **리스트 타입 정의**:
        - **Numbered**: 숫자로 매겨진 리스트 (`1.`, `2.`, ...).
        - **Bulleted**: 점(Bullet) 형태의 리스트 (`-` 또는 `*`).
        - **Plain**: 기호 없는 단순 나열 (아이템 간 개행 처리).

## [Round 2] Export Modal 고도화 및 진입점 확장 (2026-05-04 추가 기획)

### 1. Export Modal Live Preview (미리보기)
기존 Export 모달(화면 중앙)의 크기를 가로로 확장하여, 좌측에는 설정(URL, 모드 등)을 두고 우측에는 선택한 옵션(D-1, D-2 등)에 따른 **결과물 미리보기**를 제공합니다.
- **Idea I (Split Modal & Dummy Render)**: 모달을 좌/우 50:50 비율로 나눕니다. 사용자가 URL을 넣기 전이라도 임의의 더미 데이터(예: "Sample Video 1")를 사용해 현재 선택한 'Extraction Mode'가 마크다운으로 어떻게 출력될지 코드를 실시간으로 우측에 렌더링해 줍니다.
- **Idea J (Live Fetch Preview)**: URL을 넣고 파싱이 완료되면, 실제 유튜브 리스트의 첫 1~2개 항목을 바탕으로 우측에 마크다운 결과를 미리 보여줍니다. 사용자가 설정을 바꿀 때마다 즉각 반영됩니다.

### 2. 탐색기 사이드바 Context Menu 연동 (진입점 확장)
기존에는 에디터 내에서 명령어나 버튼을 통해 진입했다면, 탐색기(File Explorer)의 트리 구조와 직접 연동합니다.
- **Idea K (폴더 우클릭 -> 전용 문서 즉시 생성)**: 사이드바의 특정 폴더를 우클릭하고 `Create Youtube Playlist Note`를 클릭하면, 해당 폴더 경로가 타겟으로 고정된 상태로 모달이 열립니다. 모달에서 [Extract]를 누르면 해당 폴더 내에 `(재생목록이름).md` 파일이 즉시 생성되며 D-1(전용 문서 모드) 포맷으로 내용이 채워집니다.
- **Idea L (현재 문서 커서 위치에 삽입)**: (참고용) 탐색기 우클릭이 아닌, 열려있는 문서 내에서 우클릭 시 `Insert Youtube Playlist` 메뉴를 제공하여 현재 커서 위치에 D-2 포맷으로 리스트를 삽입하는 방식도 동시 지원합니다.

# [Phase 3] Implementation Planning
**Feature**: youtube-playlist-update
**Version**: 1
**Date**: 2026-05-04
**Agent**: frontend-developer-subagent

## 1. 개요
Expo(React Native Web/Electron) 환경에서 `youtube-playlist-update` 기능(이원화 Export 전략 및 에디터 API 기반 동기화)을 구현하기 위한 단계별 기술 설계 및 개발 계획입니다. 기능의 복잡도를 낮추기 위해 4개의 작은 Task Chunk로 분할합니다.

## 2. Task Chunks (구현 계획)

### [Task 1] 마크다운 파서 및 데이터 동기화 유틸리티 (Data Model & Parser)
- **목표**: 마크다운 본문에서 D-1(전용 문서), D-2(혼합 문서) 포맷을 파싱하고 다시 문자열로 직렬화하는 순수 함수 구현.
- **상세 작업**:
  - `utils/playlist-parser.ts` 생성
  - `parsePlaylistFromMarkdown(text, format)`: 텍스트에서 영상 데이터(URL, 제목, 노트) 추출.
  - `serializePlaylistToMarkdown(items, format)`: 조작된 객체 배열을 다시 마크다운으로 변환.
  - `diffPlaylistItems(existingItems, apiItems)`: API 결과와 비교하여 차집합(신규 영상) 도출.
- **테스트**: Jest 기반의 단위 테스트(`tests/utils/playlist-parser.test.js`) 작성하여 파싱의 무결성 검증.

### [Task 2] Slide-over Drawer UI 컴포넌트 개발 (UI Component)
- **목표**: 우측에서 부드럽게 나타나는 슬라이드 패널 및 리스트 아이템 UI 구현.
- **상세 작업**:
  - `components/ui/SlideDrawer.tsx` 생성 (React Native `Modal` 및 `Animated` 또는 `reanimated` 활용).
  - `components/playlist/PlaylistItem.tsx` 생성: 개별 영상 항목 UI (위/아래 화살표 컨트롤, 다크/퍼플 테마 적용).
- **디자인 토큰**: `constants/Theme.ts`의 Zine 900 및 Indigo~Violet 색상 적극 활용.

### [Task 3] 상태 관리 및 조작 로직 (State Management)
- **목표**: Drawer 내에서의 리스트 순서 변경, 삭제, 그리고 API 동기화 상태 관리.
- **상세 작업**:
  - `hooks/usePlaylistUpdate.ts` 커스텀 훅 개발.
  - 리스트 배열 순서 스왑(Swap) 로직.
  - 삭제 처리 및 삭제 시 노트 데이터 함께 제거 로직 구현 (Idea G).

### [Task 4] 에디터 API 연동 (Integration & Cmd+Z Support)
- **목표**: 최종 조작된 마크다운 텍스트를 안전하게 에디터에 반영하여 Undo(Cmd+Z) 지원.
- **상세 작업**:
  - 기존 에디터 컴포넌트(예: Monaco, 혹은 커스텀 에디터 훅)와 연동.
  - `fs.writeFile` 대신 에디터의 `onChange` 또는 `executeEdits` 메서드로 직렬화된 텍스트 주입.

## [Round 2] 추가 구현 계획 (2026-05-04 추가)

### [Task 5] Expanded Extraction Modal 개발 (UI Component)
- **목표**: 50:50 가로 분할 레이아웃의 추출 다이얼로그와 Live Preview 기능 구현.
- **상세 작업**:
  - `components/exporter/youtube/ExpandedExtractionModal.tsx` 생성.
  - 좌측 패널: URL 입력, Extraction Mode 선택(D-1, D-2 등), [Extract] 버튼 배치.
  - 우측 패널 (Live Preview): `isLoading` 상태에 따른 Skeleton UI 렌더링. URL 파싱 완료 시 `PlaylistParserUtils`를 이용해 가상 데이터를 렌더링.
  - 상단 탭: 마크다운 텍스트 원문(Markdown)과 시각적 렌더링(Preview) 토글 UI 구현.

### [Task 6] Explorer Context Menu 연동 (Integration)
- **목표**: 파일 탐색기 우클릭 메뉴에 단축 파이프라인 추가.
- **상세 작업**:
  - 탐색기(`components/explorer/` 하위) 우클릭 메뉴 컴포넌트에 `Create Youtube Playlist Note` 항목 추가.
  - 클릭 시 `ExpandedExtractionModal`을 호출하되, 모드를 `D-1`로 고정하고 선택된 폴더 경로(`targetDirectory`)를 상태로 전달.
  - 모달에서 [Extract] 시 `fs.writeFile`을 호출하여 지정 폴더에 직접 `.md` 파일을 생성하는 로직 연동.

### [Task 7] Interactive Preview 제어 기능 (Interaction & Clipboard)
- **목표**: `URL Mode` 및 `Text Mode` 시 미리보기 영역에서 개별 항목을 복사하고 제목을 편집할 수 있는 인터랙션 구현.
- **상세 작업**:
  - **Editable Preview List**: 우측 프리뷰 영역을 단순 `MarkdownPreview` 호출에서, 개별 `PlaylistItem`을 제어할 수 있는 리스트 컴포넌트로 분리/확장.
  - **Title Update Sync**: 제목 클릭 시 `TextInput` 노출 및 `onBlur/onEnter` 시 모달의 `previewData` 상태 업데이트 로직 연동.
  - **Copy to Clipboard**: 각 항목 우측에 `Ionicons` (copy-outline) 배치 및 `Clipboard.setString` (또는 `navigator.clipboard.writeText`) 연동.
- **의존성**: `PlaylistParserUtils.serializePlaylistToMarkdown`이 상태 변화를 즉각 반영하도록 설계.

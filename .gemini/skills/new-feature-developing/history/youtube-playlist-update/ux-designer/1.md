# [Phase 2] Appearance Designing
**Feature**: youtube-playlist-update
**Version**: 1
**Date**: 2026-05-04

## 1. UI/UX Design Goal (디자인 목표)
PO 단계에서 확정된 **Idea E (Dual Export Strategy)**를 시각적으로 매끄럽고 고급스럽게 구현하기 위한 핵심 UI를 설계합니다.
1. **Export Mode Selector (최초 추출 시)**: 사용자가 전용 문서(D-1)로 만들지, 현재 문서(D-2)에 혼합할지 직관적으로 선택하는 UI.
2. **Playlist Update Modal (업데이트 시)**: 영상 순서 변경, 삭제, 신규 영상 추가 등을 직관적으로 제어할 수 있는 조작 UI.

## 2. Design Proposals (디자인 시안)

### 시안 A: Glassmorphism 기반의 중앙 모달 (Floating Center Modal)
- **Export Selector**: 
  - 배경을 어둡게 딤(Dim) 처리하고 중앙에 반투명한 블러 효과(Glassmorphism)가 적용된 모달이 나타납니다.
  - [전용 문서 모드]와 [현재 문서 삽입 모드]가 두 개의 큰 '카드' 형태로 배치됩니다. 마우스 호버 시 은은한 보라빛 Glow 효과와 함께 각 모드가 적용되었을 때의 마크다운 예시가 미니어처 썸네일로 보여집니다.
- **Update Modal**:
  - 유튜브 리스트 아이템들이 모달 내에 세로 리스트로 나열됩니다.
  - 각 아이템 좌측에 '드래그 핸들(Drag Handle)' 아이콘이 있어, Drag & Drop으로 부드럽게(Spring 애니메이션) 순서를 변경할 수 있습니다.

### 시안 B: 우측 사이드 패널 (Slide-over Drawer) [✨ 컨텍스트 유지형]
- **Export Selector / Update Modal**:
  - 화면 중앙을 가리지 않고 우측에서 미끄러져 들어오는 패널(Drawer) 형태를 취합니다. 
  - **장점**: 사용자가 현재 작성 중인 마크다운 본문을 보면서(Context 유지) 어떤 모드로 넣을지, 영상 순서를 어떻게 바꿀지 고민할 수 있습니다.
- **조작 방식 최적화**:
  - 데스크탑 마우스뿐만 아니라 모바일/태블릿 터치 환경도 고려하여, 드래그앤드롭 외에도 우측에 직관적인 [위/아래 화살표 아이콘]을 배치합니다. 클릭 시 마이크로 애니메이션과 함께 휙휙 자리가 바뀝니다.

## 3. Visual System & Premium Palette
- **Colors (Theme-Aware)**: 
  - **Dark Mode**: 
    - Base Surface: `#18181b` (Zine 900)
    - Card/Preview: `rgba(39, 39, 42, 0.4)`
    - Text: `#f4f4f5` (Zinc 100) / Muted: `#a1a1aa` (Zinc 400)
  - **Light Mode**:
    - Base Surface: `#ffffff` (White)
    - Card/Preview: `rgba(0, 0, 0, 0.05)`
    - Preview Container: `#f9fafb` (Zinc 50)
    - Text: `#18181b` (Zinc 900) / Muted: `#71717a` (Zinc 500)
  - **Accent**: `#6366f1` ~ `#8b5cf6` (Indigo to Violet Gradient) - 공통 적용.
- **Typography**: `Inter` 폰트 사용. 모달 헤딩은 뚜렷하게(`font-weight: 700`), 설명 텍스트는 테마별 대비를 고려한 그레이 컬러로 인지 부하 감소.
- **Interaction**: 버튼 클릭 및 아이템 호버 시 크기가 미세하게 커지는(Scale 1.02) 마이크로 애니메이션 적용.
- **Theme Adaptability**: 전역 `ThemeContext`의 `isDark` 상태를 구독하여 모달 내부 모든 컴포넌트의 배경색, 텍스트색, 보더색이 즉시 전환되어야 함.

## [Round 2] Extraction Modal & Context Menu 디자인 (2026-05-04 추가)

### 1. Expanded Extraction Modal (가로 확장 다이얼로그)
- **레이아웃 확장**: 
  - 기존 중앙 다이얼로그의 너비를 **960px**, 높이를 **640px**로 확장하여 더 넓은 시각적 개방감을 확보합니다.
  - 고해상도 디스플레이 환경에서 콘텐츠가 답답해 보이지 않도록 좌/우 패널의 패딩을 **32px**로 상향 조정합니다.
- **Spacing System (8-Point Grid)**:
  - 컴포넌트 간 간격은 8px 단위를 기본으로 하며, 섹션 간 간격은 **24px~32px**를 유지하여 인지적 구분을 명확히 합니다.
  - 특히, **Playlist URL 입력창 하단에는 최소 32px 이상의 마진**을 두어 푸터 영역(API Key Status)과 시각적으로 분리합니다.
- **좌측 패널 (Settings)**: 
  - 원본 스크린샷과 동일하게 Extraction Mode (D-1/D-2), Link Format (Text/URL/Card/Video), List Type (Numbered/Bulleted/Plain), Playlist URL, Options 입력 폼을 유지합니다.
- **List Type UI**:
    - Link Format과 동일하게 3개의 토글 버튼이 나열된 Segmented Control 형태를 사용합니다.
    - 선택 시 우측 Live Preview에 즉시 반영되어야 합니다.
- **우측 패널 (Live Fetch Preview)**:
  - 배경색과 미세하게 구분되는 카드(`rgba(39, 39, 42, 0.4)`) 형태의 컨테이너를 배치합니다.
  - URL이 없거나 로딩 중일 때는 은은하게 반짝이는 **스켈레톤(Skeleton) UI**를 표시합니다.
  - 파싱이 완료되면 실제 유튜브 영상의 첫 1~2개 항목이 현재 선택된 Mode(D-1 또는 D-2)로 렌더링된 결과를 표시합니다.
  - **상단 탭**: `Markdown` / `Preview` 탭을 제공하여, 마크다운 원본 텍스트로 볼지 랜더링된 결과물로 볼지 토글할 수 있게 합니다.

### 2. File Explorer Context Menu (탐색기 우클릭 메뉴)
- **디자인 가이드**: Mark Explorer의 기본 컨텍스트 메뉴 스타일을 상속합니다.
- **메뉴 항목 구성**:
  - `Create Youtube Playlist Note` 항목 추가.
  - 직관적인 인지를 위해 메뉴 좌측에 붉은색 YouTube 아이콘(`LogoYoutube`)을 배치하여 프리미엄한 디테일을 살립니다.
  - 클릭 시 즉각적으로 Expanded Modal이 중앙에 팝업되며, Extraction Mode는 `D-1 (전용 문서)`로 강제 선택/비활성화 처리됩니다.

### 1. Expanded Extraction Modal Layout
- **구조**: 50:50 가로 분할 레이아웃, 전체 화면의 약 90% 너비 및 85% 높이 활용.
- **좌측 패널 (Settings)**:
  - Extraction Mode, Link Format, **List Type (기본값 Plain)**, Item Limit (수동 입력 필드, 기본값 20), Playlist URL 입력창.
  - **스크롤 대응**: 설정 항목이 많아질 경우를 대비해 내부 `ScrollView` 적용, 하단 `Extract` 버튼은 하단에 고정 노출되도록 배치.
- **우측 패널 (Live Preview)**:
  - 상단에 **2-Tab Navigation** 배치:
    1.  **Markdown**: 원시 마크다운 텍스트 확인.
    2.  **Preview (Honest)**: `MarkdownPreview`를 통한 **실제 결과물의 1:1 시각적 렌더링**.
  - 테마 대응 배경색: 다크(`#18181b`), 라이트(`#f9fafb`).

### 3. D-1 Header 표준 디자인
- 헤더(`##`)는 제목만 표시하여 TOC(목차) 가독성을 높입니다.
- 바로 아래 줄에 `[제목](URL)` 형태의 링크를 배치하여 제목의 하이퍼링크 기능을 유지합니다.
- 예시:
  ```markdown
  ## 영상 제목
  [영상 제목](https://youtube.com/...)
  ```

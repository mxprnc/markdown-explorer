# Git Automation Commands & Helpers

This directory contains the custom AI-powered Git helpers designed to streamline, translate, and standardize the commit workflow for the project. By analyzing staged changes (`git diff --cached`) and mapping them against user intent, these tools ensure commit history strictly adheres to the Conventional Commits specification.

---

## 🛠️ Gemini CLI Chat Commands

If you are collaborating within the Gemini CLI chat session, you can use these granular slash-commands.

*Before running these commands for the first time in a session, ensure they are loaded by typing:*
```text
/commands reload
```

### 1. `/git:commit-message [intent]`
Generates a highly compliant, English-only commit message based on staged files and saves it to a temporary location.

- **Usage**:
  ```text
  /git:commit-message docs: (README) add guides for commit helpers
  ```
- **Behavior**:
  - Checks if staged files are present. Aborts with a warning if none are detected.
  - Automatically translates multilingual user input to English and maps it to Conventional Commit headers.
  - Saves the generated draft to `./.gemini/commands/git/git-temp/<type>-<subject-kebab-case>/<id>.md` (e.g., `docs-add-guides-for-commit-helpers/001.md`).
  - Appends staged-file metadata (paths, sizes, hashes) at the bottom inside a `<!-- gemini-cli-metadata -->` comment block.
  - Returns a clickable link to the draft markdown file so you can open and edit it if needed.

### 2. `/git:commit`
Finalizes the commit using the pre-generated draft.

- **Usage**:
  ```text
  /git:commit
  ```
- **Behavior**:
  - Locates the most recent `<id>.md` draft under `./.gemini/commands/git/git-temp/`.
  - **Change Guard**: Compares current staged file hashes/sizes with the metadata saved in the draft. If staged files have changed since generating the message, it warns you and prompts whether to proceed as-is or generate a new message.
  - **Comment Stripping**: Completely strips out the `<!-- gemini-cli-metadata -->` HTML block before executing the commit, keeping only the pure, polished commit message.
  - **Cleanup**: Automatically deletes the associated temporary folders under `./.gemini/commands/git/git-temp/` upon successful commit.

### 3. `/git:force-commit`
An automated end-to-end command that generates the message, strips metadata, and commits immediately in one go.

- **Usage**:
  ```text
  /git:force-commit
  ```
- **Behavior**:
  - Checks for staged files.
  - Generates the commit message in English under `./.gemini/commands/git/git-temp/`.
  - Removes the metadata comment blocks and commits the staged changes directly without requesting user reviews or editing prompts.
  - Safely deletes the temporary folders.

## 📜 Conventions Reference
All commit messages generated by these tools conform strictly to the standards defined in the project's [commit-message-guide.md](file:///Users/alpha300uk/workspace/0.toy-project/markdown-explorer/project/markdown-explorer-v1/.gemini/commands/git/commit-message-guide.md). 
- **Language**: English Only
- **Structure**: `<type>(<scope>): <subject>` followed by a body separated by an empty line.
- **Mood**: Imperative mood for the header (e.g. `Add` instead of `Added`/`Adds`).
